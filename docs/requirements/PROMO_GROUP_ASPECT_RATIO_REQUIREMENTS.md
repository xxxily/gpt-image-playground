# 展示组可配置展示比例需求文档

## 1. 背景

当前展示内容体系已经支持展示位、全局展示组、分享展示组和展示素材。前端展示时使用 `object-contain` 完整展示图片，避免裁切素材内容；但展示比例仍主要由展示位固定推荐值决定，例如生成区头部偏 `4:1`，顶部横幅和历史横幅在桌面端偏 `10:1`，移动端偏 `4:1`。

这种设计对早期少量固定展示位足够，但对真实投放不够灵活：

- 运营方已有物料可能是 `16:9`、`1:1`、`4:5`、`3:4`、`1.91:1` 等常见比例，当前固定比例会迫使用户重做素材。
- 同一个展示位可能承载不同投放目标，例如横幅广告、活动卡片、二维码引导、品牌图、视频封面，它们天然比例不同。
- 分享展示组经常绑定具体用户、渠道或短链，同一个展示位下不同分享 Profile 需要不同素材比例。
- 当前 PC 与移动端推荐比例不一致，用户需要为同一展示组理解两套尺寸规则，维护成本高。

因此，展示比例应从“展示位固定推荐”升级为“展示组级配置”。管理员在新建或编辑全局展示组、分享展示组时选择该组的展示比例；公共前端根据接口返回的展示组比例渲染容器，无论 PC 还是移动端都保持同一个比例。这样用户只要选择与现有物料匹配的比例，就能在不同端正常投放。

## 2. 目标

1. 全局展示组和分享展示组都支持配置展示比例。
2. 每个展示组可以拥有不同展示比例；同一个展示位下的不同展示组互不影响。
3. 同一个展示组必须只有一个 canonical 展示比例，组内所有素材、所有轮播帧、桌面图和移动图都使用这个比例展示。
4. 前端公共展示在移动端和 PC 端都按展示组比例保留展示区域，不再按设备切换另一套比例。
5. 后台提供常见比例预设，也支持管理员自定义比例。
6. 新建展示组必须选择比例；编辑历史展示组时要有清晰的兼容和迁移策略。
7. 公共接口、桌面端远程展示读取、旧环境变量展示兜底都能安全降级，不影响现有页面加载。
8. 管理后台的素材预览、比例指导、校验提示都围绕展示组比例，而不是只围绕展示位固定推荐。

## 3. 非目标

- 不在本需求中实现图片裁剪、自动扩图、智能补边或重新生成物料。
- 不要求服务端下载远程图片做强制尺寸识别；远程图片比例检测可作为后台预览的辅助校验。
- 不改变展示组优先级：分享展示组仍优先于全局展示组，全局展示组仍优先于旧环境变量展示内容。
- 不把比例做成素材级配置。素材级比例会破坏同组轮播稳定性，因此 V1 明确不支持。
- 不改变短链、分享 Profile、权限 Key 的基本语义。

## 4. 核心原则

### 4.1 比例归属

展示比例属于展示组，不属于展示位，也不属于展示素材。

- 展示位只描述页面位置、默认轮播间隔、默认切换方式和可选推荐范围。
- 展示组保存实际展示比例。
- 展示素材只保存图片 URL、可选链接、可选标题、可选替代文本、排序、权重、设备和时间窗。
- 公共接口返回当前被选中的展示组比例，前端只消费这个比例渲染。

### 4.2 同组统一

同一个展示组下必须保持完全相同的展示比例：

- 管理后台不能给单张素材单独设置比例。
- 轮播切换时容器高度不能因为不同素材而变化。
- 桌面图 URL 和移动图 URL 都放入同一个比例容器里展示。
- 如果素材图片自身比例与展示组比例不一致，前端仍使用展示组比例容器并完整显示图片；后台应提示会出现留白或视觉空间浪费。

### 4.3 跨端一致

展示组比例在 PC、移动端、Tauri 桌面、Tauri Android 中保持一致。移动端可以改变最大宽度、边距、字体和周围布局，但不能改变展示组比例本身。

## 5. 常见比例与自定义比例

### 5.1 常见比例预设

后台新建和编辑展示组时应提供以下预设，按使用场景分组展示：

| 分组 | 比例 | 典型用途 |
| --- | --- | --- |
| 方形 | `1:1` | 品牌图、头像式推荐、电商主图、二维码卡片 |
| 竖版卡片 | `4:5`、`3:4`、`2:3`、`9:16` | 移动信息流、小红书/Instagram 卡片、故事页、竖版封面 |
| 横版卡片 | `5:4`、`4:3`、`3:2`、`16:9` | 内容封面、视频封面、普通横图 |
| 分享卡片 | `1.91:1`、`2:1` | Open Graph、链接卡片、社交分享预览 |
| 横幅 | `3:1`、`4:1`、`5:1`、`10:1` | 顶部横幅、工具区小横幅、窄幅广告 |
| 超宽 | `21:9`、`12:1` | 沉浸式横幅、宽屏视觉、特殊投放 |

新建默认值：

- `generation_form_header` 新建展示组默认选中 `4:1`。
- `app_top_banner` 新建展示组默认选中 `10:1`，但允许改为任何预设或自定义比例。
- `history_top_banner` 新建展示组默认选中 `10:1`，但允许改为任何预设或自定义比例。

### 5.2 自定义比例

自定义比例应使用两个正整数输入：

```text
宽比例 : 高比例
```

规则建议：

- 宽、高必须是正整数。
- 保存时归一化为最简整数比，例如 `1920:1080` 保存为 `16:9`。
- 支持小数型常见比例的显示别名，例如 `1.91:1`；内部可保存为归一后的整数或结构化数值。
- 建议硬限制长边/短边比例不超过 `20:1`，避免异常布局；超过推荐范围时应给出明确错误。
- 对 `10:1`、`12:1` 这类超宽横幅应允许保存，因为当前顶部横幅已有此类使用场景。
- 自定义输入旁边提供实时预览框，管理员能直观看到比例形态。

## 6. 后台管理需求

### 6.1 展示组表单

新增全局展示组和新增分享展示组时，表单必须包含“展示比例”字段。

字段交互：

- 默认选中当前展示位的初始比例。
- 支持从预设列表选择。
- 支持切换到“自定义”并输入宽、高比例。
- 显示当前比例预览。
- 显示当前比例预览，不再额外给出素材尺寸建议；管理员按自己的实际物料选择比例即可。
- 显示说明：同一展示组内所有素材都会按该比例展示。

编辑展示组时：

- 可以修改展示比例。
- 修改比例前必须提示影响：组内所有素材预览和前端展示都会使用新比例，可能出现留白或视觉变小。
- 如果展示组已有素材，保存后应保留素材 URL，不自动改图、不裁图。
- 如果比例从旧值改为新值，应写入审计日志。

### 6.2 展示组列表

全局展示组和分享展示组列表应新增“比例”列或在展示位信息旁展示比例。

列表至少展示：

- 比例标签，例如 `16:9`、`4:5`、`自定义 7:3`。
- 是否为自定义比例。
- 对历史未迁移组显示“继承展示位比例”或迁移后的默认比例，不能让管理员误以为已经手动配置过。

### 6.3 素材管理页

素材管理页必须围绕展示组比例更新：

- 新增素材表单顶部展示当前展示组比例，而不是展示素材尺寸建议。
- 桌面图 URL 和移动图 URL 至少填写一个；只填一个时另一端沿用同一张图。
- 标题、替代文本和点击链接都允许为空；点击链接为空时前端只展示图片，不渲染可点击跳转。
- 素材预览框使用展示组比例。
- 素材列表缩略图使用展示组比例，避免后台看到的预览和前端不一致。
- 上传或填写图片 URL 后，如浏览器能读取图片 `naturalWidth/naturalHeight`，应计算素材实际比例并与展示组比例比较。
- 比例偏差在容忍范围内时显示通过；偏差较大时显示警告，例如“该图约为 1:1，展示组为 16:9，前端会完整显示但两侧或上下会留白。”
- 比例检测不能成为远程图片不可读取时的硬阻塞；无法读取时给出“无法自动检测，请人工确认”的提示。

### 6.4 文案与国际化

新增或变更的后台可见文案必须进入 `src/lib/i18n/*` 现有翻译体系，至少保持中文和英文同步。比例值、URL、Profile ID、技术标识使用 `data-i18n-skip`。

## 7. 数据模型与接口需求

### 7.1 推荐字段

建议在 `promo_configs` 增加展示组级比例字段：

```typescript
type PromoAspectRatioSource = 'preset' | 'custom' | 'legacySlot';

type PromoConfigAspectRatio = {
    aspectRatioWidth: number;
    aspectRatioHeight: number;
    aspectRatioLabel: string;
    aspectRatioSource: PromoAspectRatioSource;
};
```

SQLite 字段建议：

- `aspectRatioWidth INTEGER`
- `aspectRatioHeight INTEGER`
- `aspectRatioLabel TEXT`
- `aspectRatioSource TEXT`

也可以使用单个 JSON 字段，但结构化列更利于校验、列表展示和未来筛选。

### 7.2 校验规则

服务端创建和更新展示组时必须校验：

- 新建展示组必须有有效比例。
- 宽、高为正整数。
- 比例归一化后保存，避免 `1920:1080` 和 `16:9` 被视为不同值。
- `aspectRatioSource` 只能为允许枚举。
- `legacySlot` 只允许历史数据或迁移兼容路径使用，新建表单不能主动创建 `legacySlot`。
- 更新展示位、启停展示组、调整轮播间隔时不能丢失比例字段。

### 7.3 公共接口

`/api/promo/placements` 返回的每个 `PromoPlacement` 应增加比例信息：

```typescript
type PromoPlacement = {
    slotKey: string;
    slotName: string;
    enabled: boolean;
    intervalMs: number;
    transition: PromoTransition;
    source: PromoPlacementSource;
    aspectRatio: {
        width: number;
        height: number;
        label: string;
        source: 'preset' | 'custom' | 'legacySlot';
    };
    items: PromoPlacementItem[];
};
```

前端消费规则：

- 如果接口返回 `aspectRatio`，优先使用该比例。
- 如果接口没有返回 `aspectRatio`，按旧逻辑使用展示位默认比例，保证旧服务端、桌面远程服务或缓存响应不崩。
- 旧环境变量展示内容继续使用 `generation_form_header` 的旧默认比例，除非未来扩展环境变量配置。

### 7.4 能力接口

`/api/promo/capabilities` 应返回比例预设和自定义比例约束，便于后台表单和未来外部管理端复用：

```typescript
type PromoAspectRatioPreset = {
    id: string;
    label: string;
    width: number;
    height: number;
    group: 'square' | 'portrait' | 'landscape' | 'shareCard' | 'banner' | 'ultraWide';
    recommendedSlots?: string[];
};
```

## 8. 前端展示需求

### 8.1 渲染规则

`PromoCarousel` 应使用展示组比例作为 `aspect-ratio`：

- 所有轮播帧共用同一个比例容器。
- 图片继续使用 `object-contain`，不裁切。
- 轮播控制按钮不能挤压比例区域导致比例变化；如果当前按钮在容器外部，需要确认总布局不会让图片区域高度跳变。
- 页面顶部、历史顶部、生成区头部都应支持任意合法比例。

### 8.2 响应式约束

为了保持可用性，前端可以对不同展示位设置最大宽度、最大高度或周围留白，但不能改变比例本身。

建议：

- `generation_form_header` 空间较小，遇到 `1:1`、`4:5`、`9:16` 等高比例时应允许表单区域换行或降级到独立行展示，避免挤压标题和主要操作。
- 顶部横幅和历史横幅可以按容器宽度自然计算高度，但应设置合理的最大高度，防止极端自定义比例撑开首屏。
- 移动端必须检查安全区、窄屏宽度和长文本，不得出现展示内容压住表单控件或历史面板标题。

### 8.3 降级

- 请求展示内容失败时维持现有逻辑：使用可用缓存或隐藏展示内容，不影响主功能。
- 接口返回无效比例时丢弃比例并使用展示位默认比例，同时可在开发日志中记录。
- 桌面端配置远程展示服务时，如果远端仍是旧版本，客户端按旧比例展示。

## 9. 兼容与迁移

### 9.1 历史数据

历史展示组没有比例字段，迁移需要避免突然改变线上视觉。

建议分两阶段：

1. 数据库新增可空比例字段，公共接口对空值返回旧展示位比例逻辑。
2. 后台编辑历史展示组时要求管理员确认并保存一个展示组比例。

如需要一次性填充默认值，建议：

- `generation_form_header` 填充 `4:1`。
- `app_top_banner` 填充 `10:1`。
- `history_top_banner` 填充 `10:1`。

但该方案会让移动端从原先部分场景的 `4:1` 变为单一 `10:1`，需要作为明确产品变更处理。更稳妥的 V1 是让历史组保持 `legacySlot` 兼容，新增组走展示组比例。

### 9.2 分享链接与短链

分享链接中的 `promoProfileId` 不需要增加比例参数。比例跟随后台展示组：

- 管理员调整分享展示组比例后，所有使用该 Profile 的长链接和短链下次访问立即生效。
- 短链绑定指定 Profile 时不需要感知比例。
- 如果分享展示组失效并降级到全局展示组，前端使用全局展示组自己的比例，而不是保留分享展示组比例。

### 9.3 缓存

前端展示内容缓存 key 当前包含端点、展示位、surface、device、Profile ID。新增比例后无需单独加入 key，因为比例随 placement payload 一起返回；但后台更新展示组比例后，用户可能在缓存 TTL 内看到旧比例。

可选优化：

- 缩短后台保存后的本地预览缓存。
- 公共接口响应增加 `updatedAt` 或 `version`，未来用于更精细的客户端缓存失效。

## 10. 验收标准

### 10.1 后台

- 新建全局展示组时必须能选择预设比例或自定义比例。
- 新建分享展示组时必须能选择预设比例或自定义比例。
- 每个展示组列表项能看到当前比例。
- 编辑展示组比例后，素材管理页预览立即使用新比例。
- 同一展示组内无法给不同素材保存不同展示比例。
- 自定义比例输入非法值时不能保存，并展示明确错误。
- 修改比例、创建展示组、更新展示组都写入现有审计体系。

### 10.2 公共展示

- 同一展示组多张素材轮播时，容器高度不随素材切换变化。
- 分享展示组和全局展示组可以在同一个展示位使用不同展示比例。
- 分享展示组失效降级到全局展示组时，使用全局展示组比例。
- PC 和移动端访问同一个展示组时，展示区域比例一致。
- 图片完整展示，不因比例变化被裁切。
- 无比例字段或无效比例响应不会导致页面崩溃。

### 10.3 兼容

- 历史展示组在未编辑前不应因为迁移突然破坏线上展示。
- 旧环境变量 `generation_form_header` 兜底仍可显示。
- Tauri 桌面端读取旧版远程展示接口时仍能展示。
- 移动端窄屏下展示内容不遮挡主要生成、编辑、历史操作。

## 11. 测试计划

### 11.1 单元测试

- 比例解析：`16:9`、`1.91:1`、`1920:1080`、空值、负数、小数、极端值。
- 比例归一化：`1920:1080 -> 16:9`、`1200:300 -> 4:1`。
- 预设查找和默认值：不同展示位返回对应初始比例。
- 公共接口构造：share/global/legacy 三种来源的 `aspectRatio` 行为。
- 历史空比例 fallback 行为。

### 11.2 集成测试

- `/api/admin/promo/configs` 创建和更新比例字段。
- `/api/promo/placements` 对分享展示组优先、全局展示组兜底、分享组失效降级时的比例返回。
- `/api/promo/capabilities` 返回比例预设和约束。
- 审计日志包含比例变更摘要。

### 11.3 UI 与视觉验证

- 管理后台新建全局展示组、新建分享展示组、编辑比例、管理素材预览。
- 前台三个展示位在 light/dark 主题下展示 `1:1`、`4:5`、`16:9`、`4:1`、`10:1`、自定义比例。
- 移动端和桌面端检查同组比例一致。
- 生成区头部在高比例素材下不挤压标题、按钮或输入区域。
- 轮播控制按钮在多素材时不遮挡图片内容。

## 12. 实施顺序建议

1. 新增比例常量、解析/归一化/校验工具和单元测试。
2. 扩展 `promo_configs` schema、SQLite 创建语句和迁移语句。
3. 扩展后台创建/更新 schema、服务端 create/update/list 逻辑和审计日志。
4. 扩展 `/api/promo/capabilities` 和 `/api/promo/placements` 返回结构，并保留旧响应 fallback。
5. 改造展示组表单、展示组列表、素材管理页预览和比例检测提示。
6. 改造 `PromoCarousel` 使用 placement aspect ratio，调整各展示位 wrapper 的响应式约束。
7. 补齐 Web、Tauri 桌面远程服务、移动端窄屏的验证。
8. 功能上线后再更新用户手册 `docs/展示内容与后台管理使用手册.md`，避免在需求阶段提前宣称已发布能力。
