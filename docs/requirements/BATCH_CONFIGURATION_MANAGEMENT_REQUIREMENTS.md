# 批量功能独立配置与规划方式管理需求文档

## 1. 背景与问题

现有批量规划能力复用了提示词润色的模型选择、提示词配置和部分调用链路。这在技术上减少了早期实现成本，但在产品心智上容易造成混乱：

- 批量规划模型混在提示词润色模型配置里，用户不清楚它到底服务于“润色当前 prompt”还是“拆分/规划批量任务”。
- 批量规划提示词、参数润色提示词和自定义润色提示词放在同一配置语境下，容易被误改。
- 用户想配置批量任务数量、规划方式、默认规划提示词、模型或二次调整策略时，没有一个明确入口。
- 后续要扩展更多批量规划方式时，如果继续挂在润色配置下，会让设置页语义越来越不清晰。

因此，批量功能需要从提示词润色配置中拆出来，形成独立的 **批量配置** 管理区域。提示词润色只负责单条 prompt 的改写；批量配置负责把输入内容、源图和用户目标规划成多个可执行任务。

相关既有需求：

- [批量生图与任务规划需求文档](./BATCH_IMAGE_GENERATION_REQUIREMENTS.md)
- [批量任务手动切分与 JSON 导入需求文档](./BATCH_TASK_MANUAL_SPLIT_AND_JSON_IMPORT_REQUIREMENTS.md)
- [提示词润色模型选择重构需求文档](./PROMPT_POLISH_MODEL_SELECTION_REFACTOR_REQUIREMENTS.md)

## 2. 目标

1. 新增独立的 **批量配置** 设置入口，不再把批量规划默认模型和默认提示词混放在 **提示词润色配置** 中。
2. 批量配置支持选择和管理批量规划模型，模型来源继续复用统一供应商端点与模型目录。
3. 批量配置支持参数润色配置，用于控制批量任务生成前如何对每条子任务 prompt 或共享参数进行规范化。
4. 批量配置支持规划方式管理，默认包含 `AI 自动`、`内容拆分`、`多版本探索`、`参考图变体`、`文本切分`、`JSON 导入`。
5. 批量配置支持查看、编辑、恢复默认规划提示词，尤其是默认 `AI 自动` 规划方式的系统提示词。
6. 规划方式应具备可扩展结构，后续可加入自定义规划方式、批量模板、垂直场景预设或外部导入策略。
7. 用户在批量面板里看到的是批量任务语境，不需要理解提示词润色配置才能完成批量规划。
8. Web 与 Tauri 桌面端使用同一份配置语义，不能引入只在 Web 服务端可用的配置路径。

## 3. 非目标

- 不移除提示词润色功能。
- 不把批量规划模型自动继承为提示词润色模型，也不把提示词润色模型自动继承为批量规划模型。
- 不新增批量专用 API Key、Base URL 或独立供应商表；端点和凭证仍归统一供应商端点管理。
- 不在 P0 实现第三方插件式规划引擎市场。
- 不要求 P0 支持每个规划方式绑定完全不同的网络协议；首版仍以 OpenAI/OpenAI 兼容和 Anthropic/Anthropic 兼容文本规划模型为主。
- 不允许批量配置存储或导入 API Key、密码、分享密钥等敏感信息。

## 4. 产品边界

### 4.1 提示词润色配置只管理单条 prompt

提示词润色配置应保留以下职责：

- 提示词润色默认模型。
- 润色思考参数。
- 自定义润色提示词。
- 润色下拉顺序和相关展示偏好。

提示词润色配置不再展示：

- 批量规划默认模型。
- 批量规划默认系统提示词。
- 批量规划方式。
- 批量任务数量策略。
- 批量子任务参数润色策略。

如果旧版本设置中已经把批量规划模型展示在润色配置里，需要迁移到批量配置页展示。底层字段可以继续使用统一任务默认值字段，但 UI 归属必须拆开。

### 4.2 批量配置管理批量工作流

批量配置应管理以下职责：

- 批量规划模型选择。
- 批量参数润色设置。
- 默认规划方式。
- 各规划方式的提示词查看与管理。
- 批量任务数量默认值与安全上限。
- 批量预览、二次调整和确认前校验的默认行为。
- 规划方式启用、排序和可见性。

批量配置不是供应商端点管理。添加端点、填写 API Key、配置 Base URL、刷新模型列表等底层连接能力仍在供应商端点管理中完成；批量配置只选择可用于批量规划的任务默认模型。

## 5. 设置入口与信息架构

建议在设置中形成三个清晰入口：

| 设置入口 | 职责 |
| --- | --- |
| 供应商端点管理 | 管理端点、凭证、Base URL、模型发现和模型白名单 |
| 提示词润色配置 | 管理单条 prompt 润色模型、润色提示词和思考参数 |
| 批量配置 | 管理批量规划模型、规划方式、默认规划提示词和批量参数润色 |

批量配置建议包含以下分区：

1. **模型与调用**
   - 批量规划模型。
   - 调用模式和兼容性提示。
   - 未配置模型时的空状态和添加端点入口。
2. **规划方式**
   - 默认规划方式。
   - 启用/禁用规划方式。
   - 规划方式排序。
   - 规划方式说明和适用场景。
3. **默认规划提示词**
   - 查看内置默认提示词。
   - 编辑自定义覆盖提示词。
   - 恢复默认。
   - 校验输出 JSON 契约说明。
4. **参数润色**
   - 是否在创建预览前润色子任务 prompt。
   - 是否润色共享参数说明。
   - 润色强度或策略。
   - 是否保护用户手工锁定条目。
5. **数量与安全**
   - `AI 自动` 最大任务数。
   - 固定数量默认值。
   - 确认前二次提醒阈值。
   - 单次预览最大条目数。

## 6. 批量模型选择

### 6.1 模型来源

批量规划模型必须来自统一供应商端点与模型目录：

- 允许 OpenAI/OpenAI 兼容端点。
- 允许 Anthropic/Anthropic 兼容端点。
- 端点可以来自官方供应商，也可以来自第三方兼容中转。
- 禁用端点不能出现在可选列表中。
- 非 OpenAI/Anthropic 兼容族端点首版不进入批量规划模型选择。

保存字段继续使用统一任务默认值：

```ts
modelTaskDefaultCatalogEntryIds['prompt.batchPlan']
```

批量规划模型与提示词润色模型独立：

- 选择批量规划模型不改变 `modelTaskDefaultCatalogEntryIds['prompt.polish']`。
- 选择提示词润色模型不改变 `modelTaskDefaultCatalogEntryIds['prompt.batchPlan']`。
- 用户可以显式选择同一个模型作为两个任务的默认值，但必须是明确动作。

### 6.2 选择交互

批量配置中的模型选择应复用通用端点模型绑定组件：

1. 选择供应商端点。
2. 点击模型状态框。
3. 打开模型选择弹窗。
4. 自动读取该端点最新模型列表。
5. 用户单选一个模型并确认。
6. 系统写入或更新 `modelCatalog`、端点 `modelIds` 和 `modelTaskDefaultCatalogEntryIds['prompt.batchPlan']`。

如果没有可用端点，批量配置显示空状态：

- 说明需要先添加 OpenAI 兼容或 Anthropic 兼容端点。
- 提供 **添加端点** 入口。
- 点击后进入供应商端点管理，并预选通用 OpenAI 兼容端点模板。

如果端点不支持 `/models` 或模型读取失败，应允许用户手动添加隐藏模型 ID，并直接绑定为批量规划模型。

### 6.3 执行要求

批量规划执行时必须显式解析 `prompt.batchPlan` 任务默认模型。未选择批量规划模型时：

- 不应自动使用提示词润色模型。
- 不应发起远端请求。
- 批量面板应提示用户先配置批量规划模型。
- 可以提供 **使用提示词润色模型作为批量模型** 的显式快捷动作；点击后仍要写入 `prompt.batchPlan`，而不是运行时隐式继承。

## 7. 规划方式管理

### 7.1 内置规划方式

P0 至少支持以下规划方式：

| 规划方式 | 说明 | 是否调用 AI |
| --- | --- | --- |
| AI 自动 | AI 判断输入更适合内容拆分、多版本探索、参考图变体或混合规划 | 是 |
| 内容拆分 | 把长文本、段落、脚本、卖点或列表拆成多个任务 | 是 |
| 多版本探索 | 围绕同一需求生成多个风格、构图、场景或受众方向 | 是 |
| 参考图变体 | 基于当前源图生成多个参考图变体方案 | 是 |
| 文本切分 | 按换行、空行段落或分隔符确定性切分文本 | 否 |
| JSON 导入 | 导入外部工具生成的批量任务 JSON | 否 |

设置中应允许：

- 选择默认规划方式，默认值为 `AI 自动`。
- 启用或禁用非核心规划方式。
- 调整展示顺序。
- 查看每种规划方式的适用说明。
- 查看 AI 类规划方式的默认提示词。

`AI 自动` 不应被禁用，因为它是新用户最容易理解的默认入口。`文本切分` 和 `JSON 导入` 可以作为不依赖 AI 的备用方式常驻展示，或在高级设置中控制显示。

### 7.2 扩展结构

规划方式建议抽象为配置对象，而不是写死在批量面板里：

```ts
type BatchPlanningStrategy = {
    id: string;
    builtIn: boolean;
    enabled: boolean;
    order: number;
    labelKey: string;
    descriptionKey: string;
    executionType: 'ai-plan' | 'local-split' | 'json-import';
    promptTemplateId?: string;
    defaultCountMode?: 'auto' | 'fixed';
    defaultMaxCount?: number;
    defaultTargetCount?: number;
};
```

后续可在不重写批量面板的情况下加入：

- 电商商品图批量方案。
- 社媒封面批量方案。
- 分镜脚本批量方案。
- 文章配图批量方案。
- 品牌视觉多版本探索。
- 用户自定义规划方式。

### 7.3 批量面板联动

批量面板打开时应读取批量配置：

- 默认选中用户配置的默认规划方式。
- 只展示已启用的规划方式。
- 使用该规划方式绑定的提示词模板。
- 使用该规划方式默认数量策略。
- 使用批量配置中的确认前安全阈值。

如果用户在当前批量面板中临时切换规划方式，不应立即改写全局默认值；只有在设置中保存，或在面板中明确点击“设为默认”时才持久化。

## 8. 默认规划提示词管理

### 8.1 提示词类型

批量配置应管理 AI 类规划方式的提示词模板：

- `AI 自动` 默认规划提示词。
- `内容拆分` 默认规划提示词。
- `多版本探索` 默认规划提示词。
- `参考图变体` 默认规划提示词。
- 二次调整提示词。
- JSON 修复提示词。

本地确定性方式不需要 AI 提示词：

- 文本切分。
- JSON 导入。

但它们仍可以有说明文案和 schema 帮助。

### 8.2 查看与编辑

每个内置提示词模板都应支持：

- 查看默认提示词。
- 复制提示词。
- 使用自定义覆盖提示词。
- 恢复默认提示词。
- 标记当前使用的是内置默认还是自定义版本。
- 显示最后修改时间。

编辑提示词时必须说明输出契约。批量 AI 规划必须返回结构化 `BatchPlan` JSON，不能只返回自然语言列表。

建议提示词模板结构：

```ts
type BatchPromptTemplate = {
    id: string;
    strategyId: string;
    version: number;
    builtInPrompt: string;
    customPrompt?: string;
    updatedAt?: string;
    outputContract: 'batch-plan-json';
};
```

### 8.3 默认 AI 自动提示词要求

`AI 自动` 默认提示词必须明确要求模型：

- 判断用户输入是内容拆分、多版本探索、参考图变体还是混合规划。
- 返回选择该策略的原因。
- 生成 `BatchPlan` JSON。
- 保留用户原始意图，不静默编造关键事实。
- 有源图片时说明如何继承和使用参考图。
- 给出 `warnings`，记录不确定、冲突或缺失信息。
- 控制任务数量，不为凑数强拆。
- 每条任务给出可执行 prompt、来源片段、差异方向和启用状态。

### 8.4 恢复与兼容

内置提示词升级时需要兼容用户自定义覆盖：

- 用户未自定义时，自动使用最新内置提示词。
- 用户已自定义时，不自动覆盖。
- 设置页提示有新版内置提示词可查看。
- 用户可以对比自定义提示词与新版默认提示词，再决定是否恢复默认。

## 9. 参数润色配置

批量配置中的参数润色不是单条 prompt 润色配置的复制。它服务于批量预览生成后的规范化和一致性处理。

### 9.1 适用阶段

参数润色可发生在两个阶段：

1. **预览生成前**：把用户输入、统一前缀、统一后缀和源图意图整理成更清楚的规划上下文。
2. **预览生成后**：对每条子任务 prompt 做轻量规范化，使批量任务风格、格式和参数表达更一致。

P0 建议优先支持预览生成后的轻量规范化，避免在规划前过度改写用户原始输入。

### 9.2 可配置项

建议提供以下配置：

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| 启用批量参数润色 | 关闭 | 默认不额外改写 AI 规划结果 |
| 润色范围 | 子任务 prompt | 可扩展到 negative prompt、标题、备注 |
| 润色强度 | 轻量 | 只修正格式、重复和明显不完整表达 |
| 保留手工修改 | 开启 | `lockedByUser` 条目不被覆盖 |
| 保留来源片段 | 开启 | 不改写 `sourceExcerpt` |
| 保留参考图约束 | 开启 | 不删除源图继承和主体保留要求 |

参数润色如果调用模型，首版可以复用批量规划模型，也可以在后续扩展为独立的批量润色模型。P0 不建议再新增第三个模型选择，避免设置复杂度过高。

### 9.3 约束

- 参数润色不能改变任务数量。
- 参数润色不能删除用户手工锁定的条目。
- 参数润色不能把禁用条目重新启用。
- 参数润色不能移除源图继承策略。
- 参数润色不能把 JSON 导入的外部元数据写入执行参数。
- 参数润色结果需要经过同一套 `BatchPlan` 结构校验。

## 10. 配置持久化与迁移

### 10.1 建议配置结构

建议新增批量配置对象，挂在现有应用配置中：

```ts
type BatchFeatureConfig = {
    defaultStrategyId: string;
    strategies: BatchPlanningStrategy[];
    promptTemplates: BatchPromptTemplate[];
    maxAutoTaskCount: number;
    defaultFixedTaskCount: number;
    confirmLargeBatchThreshold: number;
    maxPreviewTaskCount: number;
    parameterPolish: BatchParameterPolishConfig;
};

type BatchParameterPolishConfig = {
    enabled: boolean;
    scope: 'task-prompts' | 'task-prompts-and-negative-prompts';
    intensity: 'light' | 'standard';
    preserveLockedItems: boolean;
    preserveSourceExcerpt: boolean;
    preserveSourceImagePolicy: boolean;
};
```

模型绑定继续使用：

```ts
modelTaskDefaultCatalogEntryIds['prompt.batchPlan']
```

这样可以避免批量配置重复保存端点、凭证和模型目录。

### 10.2 默认值

首次打开或旧配置迁移后，默认值建议为：

| 字段 | 默认值 |
| --- | --- |
| `defaultStrategyId` | `auto` |
| `maxAutoTaskCount` | `8` |
| `defaultFixedTaskCount` | `4` |
| `confirmLargeBatchThreshold` | `100` |
| `maxPreviewTaskCount` | `30` |
| `parameterPolish.enabled` | `false` |
| `parameterPolish.intensity` | `light` |

具体数值可以在实现时根据现有任务队列和移动端性能再调整，但需要保持有界，避免用户误创建过大批次。

### 10.3 迁移规则

旧版本配置迁移要求：

- 如果已存在 `modelTaskDefaultCatalogEntryIds['prompt.batchPlan']`，继续保留。
- 如果旧 UI 中曾显示批量规划模型，迁移后只在批量配置中展示。
- 不从 `prompt.polish` 自动复制到 `prompt.batchPlan`。
- 如果旧配置没有批量配置对象，按默认值补齐。
- 未知规划方式、未知提示词模板或异常字段应归一化为安全默认值。
- 导入配置和云同步配置时同样执行归一化。

## 11. 批量面板体验要求

批量面板需要根据独立批量配置调整：

- 面板顶部展示规划方式，而不是隐藏在润色配置语境里。
- 未配置批量规划模型时，AI 类规划方式显示可理解的配置提示。
- 文本切分和 JSON 导入不依赖批量规划模型，仍可使用。
- 默认规划方式来自批量配置。
- 每个规划方式展示自己的说明和数量选项。
- AI 类规划方式支持查看当前使用的提示词摘要，但详细编辑仍进入批量配置。
- 生成预览后，继续复用统一预览、逐条编辑、禁用、重排和确认入队能力。

## 12. UI 与国际化要求

所有新增或调整的用户可见文案必须进入 `src/lib/i18n/*`，并保持每种支持语言同步。

建议新增或复用以下语义：

- 批量配置。
- 批量规划模型。
- 默认规划方式。
- 规划方式管理。
- 默认规划提示词。
- 查看默认提示词。
- 使用自定义提示词。
- 恢复默认提示词。
- 参数润色。
- 轻量润色。
- 保留手工修改。
- 批量安全上限。
- 使用提示词润色模型作为批量模型。

布局要求：

- 移动端单列可滚动。
- 桌面端可分区展示，但不能使用嵌套卡片堆叠过深。
- 明暗主题都要检查。
- 关键操作不依赖 hover。
- 不使用 `window.alert`、`window.prompt` 或 `window.confirm`。
- 长提示词模板需要折叠、展开、复制和编辑状态。

## 13. 安全与性能

- 批量配置不得保存 API Key、密码、分享令牌或其他敏感信息。
- 导入导出默认不包含 API Key，继续遵守现有密钥导出规则。
- 分享链接不应携带批量配置中的敏感字段；批量提示词模板可以分享，但端点凭证不能分享。
- 批量提示词模板和预览草稿需要体积限制，不能无限写入 localStorage。
- 规划方式列表和提示词模板加载不能阻塞首屏。
- 批量面板生成预览时必须保留现有并发、超时和取消能力。
- 不在热路径上对长文本做重型 hash 或 Blob 扫描。

## 14. 验收标准

### 14.1 配置拆分

- 设置中存在独立 **批量配置** 入口。
- **提示词润色配置** 不再展示批量规划模型、批量规划提示词或批量规划方式。
- 用户可以在批量配置中选择批量规划模型。
- 只配置提示词润色模型时，批量 AI 规划仍提示需要选择批量规划模型。
- 用户显式点击使用同一个模型后，系统写入 `prompt.batchPlan`。

### 14.2 规划方式

- 批量配置可以选择默认规划方式。
- 批量面板打开时默认选中用户配置的规划方式。
- `AI 自动`、`内容拆分`、`多版本探索`、`参考图变体`、`文本切分`、`JSON 导入` 的职责清晰可见。
- 文本切分和 JSON 导入在未配置 AI 模型时仍可生成预览。

### 14.3 提示词管理

- 用户可以查看 `AI 自动` 默认规划提示词。
- 用户可以为 AI 类规划方式设置自定义提示词。
- 用户可以恢复内置默认提示词。
- 自定义提示词不会在内置提示词升级时被静默覆盖。
- AI 规划请求仍返回并校验结构化 `BatchPlan` JSON。

### 14.4 参数润色

- 批量配置中存在参数润色开关。
- 默认关闭，不改变当前批量规划输出。
- 开启后不会改变任务数量、启用状态、源图继承策略或手工锁定条目。
- 润色结果经过同一套结构校验，失败时保留原预览并展示错误。

### 14.5 兼容与同步

- 旧配置导入后不会丢失 `prompt.batchPlan` 模型绑定。
- 云同步保留批量配置和任务默认模型绑定。
- 默认导出和分享不泄露 API Key。
- Web 与 Tauri 桌面端批量配置语义一致。
