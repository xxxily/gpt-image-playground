# 素材灵感库与资源物料仓库分区工作台需求细化

## 1. 背景与问题

现有素材灵感库和资源物料仓库已经规划并初步实现为全局浮动菜单入口 + 可复用 Drawer 承载。Drawer 对“临时查看、管理配置、添加网站、导入导出”等短任务是合适的，但对“持续浏览灵感并立即使用”的创作流程存在明显阻断：

1. 用户在 Drawer 内打开灵感网站后，主工作台被覆盖，无法同时看到提示词输入区、源图区域和生成按钮。
2. 用户在第三方灵感网站复制图片后，需要先关闭 Drawer，再回到工作台粘贴源图和输入提示词，流程被打断。
3. 桌面端屏幕空间足够，如果只能覆盖式打开，用户实际会更倾向于外部浏览器 + 系统分屏，应用内灵感库的效率优势会被削弱。
4. 资源物料仓库也有类似问题：用户选择一个素材后，常常还要继续调整提示词、对照输出效果、再选另一个素材，不应该每次使用后都关闭面板。

因此需要在保留 Drawer 的同时，新增一种 **非阻塞式分区工作台** 能力：左侧继续展示原工作台，右侧展示素材灵感库或资源物料仓库，用户可以在同一个页面内完成浏览、复制、粘贴、选素材、输入提示词、生成和查看结果。这个能力不能只做成一次性的右侧面板，而应按类似 VS Code 的工作台分区能力规划：面板之间可以左右拖动调整大小，可以折叠和恢复，并能为后续更多工作区块复用。

## 2. 核心结论

1. **Drawer 不应废弃**：它仍然适合低频管理、设置、导入导出、站点维护、失败详情和移动端全屏使用。
2. **桌面端连续浏览默认走分区工作台**：素材灵感网站、资源物料列表这类会长时间占用用户注意力的功能，在桌面端应优先以右侧分区面板打开。
3. **左右拖动调整大小是 P0 核心能力**：分区面板不是固定宽度侧栏，用户必须能像 VS Code 一样拖动分隔线调整主工作台和辅助面板占比。
4. **分区布局必须是通用能力**：不能只为素材灵感库硬写。后续批量中心、任务详情、同步活动、模型配置、提示词模板等功能也可能作为左/右/底部工作区块打开。
5. **分区面板不是 Modal**：不能使用带遮罩、焦点陷阱和 `aria-modal` 的 Dialog/Drawer 模式，否则仍会阻止主工作台操作。
6. **移动端保持全屏 Drawer**：小屏没有足够空间并排浏览和生成，移动端仍应使用全屏 Drawer，并优化“复制后返回工作台”的路径。

## 3. 目标

1. 新增可复用的 **工作台分区布局组件**，支持桌面端将当前页面切分为主工作台区和右侧辅助面板，并预留后续左侧、底部和多面板组合能力。
2. 全局浮动功能菜单支持同一个功能声明不同打开方式：分屏打开、抽屉打开、外部打开。
3. 素材灵感库在桌面端默认以分屏面板打开，用户仍可手动切换为 Drawer 或外部标签页。
4. 资源物料仓库在桌面端支持以分屏面板打开，用户选中素材并发送到编辑区后，面板默认保持打开。
5. 分区面板宽度必须支持左右拖动调整、键盘调整、双击恢复默认、收起、关闭，并记住用户偏好。
6. 主工作台在分屏状态下仍可完整操作：粘贴源图、编辑提示词、调整参数、发起任务、查看输出和操作历史。
7. 分屏状态下的粘贴流程要明确处理 iframe 限制：不能承诺从第三方 iframe 自动读取内容，但要提供可用的粘贴入口和降级路径。
8. 浅色/深色主题、桌面/移动布局、Web/Tauri 桌面/Tauri Android 都要有明确行为。

## 4. 非目标

- 不通过代理、注入脚本或读取 iframe DOM 来绕过第三方网站限制。
- 不实现真正的浏览器级多窗口管理，也不接管操作系统分屏能力；本需求关注应用内部工作台分区。
- 不在移动端强行做左右分屏。
- 不把分屏面板做成固定常驻侧边栏；它应由用户从浮动菜单或功能入口主动打开。
- 不把所有管理功能都铺在分屏首屏。低频维护仍通过按钮、Popover、Dialog 或 Drawer 二级页面进入。

## 5. 打开方式分层

后续功能入口应统一支持三类展示表面：

| 表面 | 适用场景 | 是否阻塞主工作台 | 典型功能 |
| --- | --- | --- | --- |
| 分屏面板 `split` / `docked` | 长时间参考、频繁选择、边看边做 | 不阻塞 | 素材灵感网站、资源物料仓库、任务详情、提示词模板 |
| Drawer `drawer` | 短任务、管理、配置、低频维护 | 阻塞或半阻塞 | 添加网站、导入导出、设置、详情表单 |
| 外部打开 `external` | 第三方站点不允许 iframe、需要登录或大页面浏览 | 不在应用内承载 | Behance、Pinterest 等外部网页 |

推荐默认策略：

| 功能 | 桌面端默认 | 移动端默认 | 说明 |
| --- | --- | --- | --- |
| 素材灵感库站点目录 | 分屏面板 | 全屏 Drawer | 目录和站点打开入口可长期停留 |
| 灵感网站 iframe | 分屏面板 | 全屏 Drawer | iframe 失败时提供外部打开 |
| 资源物料仓库列表 | 分屏面板 | 全屏 Drawer | 选择素材后不自动关闭 |
| 素材/网站管理 | Drawer 或 Dialog | 全屏 Drawer | 低频维护，不占主工作区 |
| 导入导出向导 | Drawer 或 Dialog | 全屏 Drawer | 有明确开始和结束 |

## 6. 工作台分区布局能力

这个能力后续会成为更完整工作台的基础，因此不能只按“右侧打开一个素材面板”实现。建议抽象成 **应用内部工作台分区布局系统**，第一阶段只启用主工作台 + 右侧辅助区，但数据模型和组件边界要能支撑更多分区。

### 6.1 布局原则

- 主工作台永远是 primary pane，负责提示词、源图、参数、生成输出和任务操作。
- 素材灵感库、资源物料仓库等是 auxiliary pane，默认停靠在右侧。
- 分区之间通过 resize handle 分隔，用户可以左右拖动调整大小。
- 分区调整不能触发主工作台状态重置，也不能导致 iframe 或素材列表无意义重载。
- 每个分区必须有最小尺寸、最大尺寸和折叠尺寸，避免拖到不可用状态。
- 布局状态应可持久化，但必须在每次恢复时重新按当前视口 normalize。
- 后续可以扩展为左侧资源树、右侧参考区、底部任务/日志区，但 P0 只实现一个右侧辅助区。

### 6.2 类 VS Code 的拖拽调整

P0 必须支持左右拖动调整主工作台和右侧辅助面板宽度，行为建议对齐 VS Code / IDE 类工作台：

- 分隔线命中区域不能太窄，视觉线可以是 1px，但可拖动热区建议 8-12px。
- 悬停分隔线时显示 `col-resize` 光标，触摸设备使用足够大的拖动热区。
- 拖动时实时更新 CSS 变量或 grid track，不应在每一帧重建业务组件。
- 拖动过程中可显示轻量参考线或激活态，帮助用户理解正在调整哪个边界。
- 拖动到最小宽度附近时可吸附到折叠状态；再次点击或拖出可恢复。
- 双击分隔线恢复默认比例，例如右侧 `min(520px, 36vw)`。
- `Esc` 在拖动中取消本次调整，恢复拖动前尺寸。
- 如果 iframe 位于面板内，拖动时可以临时覆盖透明层捕获 pointer，避免 iframe 吃掉拖拽事件。

尺寸约束建议：

| 项 | 建议值 |
| --- | --- |
| 右侧辅助区默认宽度 | `min(520px, 36vw)` |
| 右侧辅助区最小宽度 | `360px` |
| 右侧辅助区最大宽度 | `min(760px, 58vw)`，超宽屏可允许更大但不能小于主工作台最小宽 |
| 主工作台最小宽度 | `560px`，低于该值时主工作台内部降级为单列 |
| 折叠宽度 | `48px - 56px`，只展示图标、标题缩写或恢复按钮 |
| 拖动热区 | `8px - 12px` |

### 6.3 分区状态模型

建议不要只保存一个 `widthPx`。为了未来扩展，需要把布局状态拆成 pane、split 和 workspace layout：

```ts
type WorkspacePaneId = 'main' | 'right-resource' | 'left-nav' | 'bottom-activity';
type WorkspacePanePosition = 'main' | 'left' | 'right' | 'bottom';
type WorkspacePaneKind = 'main-workbench' | 'creative-resources' | 'task-center' | 'sync-activity' | 'prompt-library';

type WorkspacePaneState = {
    id: WorkspacePaneId;
    kind: WorkspacePaneKind;
    position: WorkspacePanePosition;
    active: boolean;
    collapsed: boolean;
    sizePx?: number;
    minSizePx: number;
    maxSizePx?: number;
    defaultSizePx: number;
    order: number;
};

type WorkspaceLayoutState = {
    version: 1;
    panes: WorkspacePaneState[];
    activeAuxiliaryPaneId?: WorkspacePaneId;
    lastUpdatedAt: number;
};
```

P0 可以只落地：

```text
main + right-resource
```

但接口不能把 `right-resource` 写死为素材业务，否则后续增加更多工作区块会重构成本很高。

### 6.4 尺寸计算与持久化

尺寸恢复规则：

1. 从本地偏好读取上次布局。
2. 校验 `version`、pane id、尺寸类型和数值范围。
3. 按当前 viewport 和容器宽度重新 clamp。
4. 如果主工作台会低于最小宽度，则优先缩小或折叠辅助面板。
5. 如果当前视口不支持分区，则降级为 Drawer，并保留待恢复的桌面尺寸。

持久化要求：

- 拖动过程中不要高频写 localStorage；使用 debounce 或只在 pointerup 写入。
- 保存像素值时也可以保存比例，窗口尺寸变化后优先用比例恢复，再按最小/最大值修正。
- 清空本地配置后，布局回到默认右侧辅助区宽度。
- 布局偏好不参与云同步 P0，避免不同设备屏幕尺寸互相污染；后续如果同步，也必须按设备类型保存。

### 6.5 折叠、关闭与恢复

- 关闭：移除辅助区显示，主工作台占满空间；再次从菜单打开时恢复上次尺寸。
- 折叠：辅助区缩为窄条，保留当前功能状态、iframe URL、搜索词和选中素材。
- 恢复：从折叠状态回到折叠前宽度；如果宽度不适合当前视口，按约束修正。
- 切换功能：同一个右侧辅助区内可以在素材灵感库、资源物料仓库等面板之间切换，不应创建多个互相覆盖的侧栏。
- 未来多面板：如果后续允许右侧堆叠多个工具，应使用 tabs 或 activity bar 管理，而不是同时挤出多个窄栏。

## 7. 分屏工作区体验

### 7.1 桌面端布局

当用户以分屏模式打开素材功能时，页面进入双区布局：

```text
┌───────────────────────────────────────────────────────────────┐
│ 顶部应用栏 / 设置 / 主题                                       │
├───────────────────────────────────────┬───────────────────────┤
│ 主工作台                              │ 辅助面板              │
│ - 提示词输入                          │ - 素材灵感库          │
│ - 源图粘贴/拖拽                       │ - 资源物料仓库        │
│ - 参数配置                            │ - iframe 网站         │
│ - 生成输出                            │ - 面板工具栏          │
│ - 历史记录                            │                       │
└───────────────────────────────────────┴───────────────────────┘
```

布局要求：

- 默认右侧打开，初始宽度建议 `min(520px, 36vw)`。
- 最小宽度建议 `360px`，最大宽度建议 `min(760px, 58vw)`。
- 用户必须能左右拖动分隔线调整主工作台和辅助面板大小；这属于 P0，不是后续增强项。
- 主工作台宽度不足时，现有编辑区和输出区从两列降级为单列，不能出现横向滚动和控件重叠。
- 面板高度使用可视区高度，内部独立滚动，不影响主工作台滚动。
- 分屏打开和关闭不能清空当前提示词、源图、任务状态、输出视图和历史选择。
- 用户拖动调整宽度时，应使用轻量状态更新，避免 iframe 和大列表频繁重渲染；拖动结束后再持久化。
- 用户关闭面板后，主工作台恢复当前布局；再次打开时恢复上次宽度和功能 Tab。

### 7.2 面板工具栏

分屏面板顶部应有稳定工具栏：

- 标题：当前功能名称，例如“素材灵感库”“资源物料仓库”。
- Tab 或切换：资源物料仓库 / 素材灵感库。
- 打开方式切换：分屏 / Drawer / 外部打开。
- 刷新：用于 iframe 或素材列表。
- 外部打开：第三方网站可直接用系统浏览器或新标签页打开。
- 收起：将面板折叠为窄条或图标，不丢失当前状态。
- 关闭：退出分屏。
- 更多：低频管理入口，例如管理网站、导入导出、分类管理、偏好设置。

低频操作不应默认占据面板空间，尤其是“添加网站”“分类管理”“导入导出”等，必须隐藏在按钮或二级面板中。

### 7.3 素材灵感库分屏行为

素材灵感库在分屏模式下应优先服务“找灵感并使用”的工作流：

1. 左侧主工作台保持可操作。
2. 右侧先展示站点目录、搜索、分类和常用站点。
3. 用户点击站点后，在右侧面板内打开 iframe。
4. iframe 加载失败、被 CSP/X-Frame-Options 拒绝或超时，右侧展示失败态，并提供外部打开按钮。
5. 用户在第三方页面复制图片后，可以点击左侧源图区域或“从剪贴板添加图片”按钮，将图片加入当前编辑源图。
6. 用户继续输入提示词并发起生成，右侧面板保持打开，方便继续参考。

重要限制：

- 当焦点位于 iframe 内部时，父页面通常无法接收 `paste` 事件。
- Web 端读取系统剪贴板需要用户手势和浏览器权限；不能在后台自动读取。
- Tauri 桌面可通过 `src/lib/desktop-runtime.ts` 封装系统剪贴板读取，但仍应由用户动作触发。

因此需要提供明确的粘贴入口：

- 主工作台源图区域保留或增加“从剪贴板添加图片”按钮。
- 分屏打开时，可在主工作台靠近源图区域展示轻量提示：复制素材后点击源图区或使用粘贴快捷键。
- 如果浏览器拒绝 Clipboard API，降级为提示用户点击源图上传区后按粘贴快捷键。
- 不在 iframe 中注入脚本，也不监听第三方页面内部复制行为。

### 7.4 资源物料仓库分屏行为

资源物料仓库在分屏模式下应优先服务“选素材并继续创作”的工作流：

- 素材列表、分类、搜索、标签筛选是首屏核心。
- 详情编辑、分类维护、导入导出、批量管理放在二级入口。
- 素材卡片提供主操作：发送到编辑、替换源图、收藏。
- 用户发送素材到编辑区后，面板默认保持打开，并更新素材的最近使用时间。
- 支持多选后批量发送到编辑区，但要遵守当前模型源图数量上限。
- 如果用户拖拽素材卡片到左侧源图区域可行，可以作为增强；P0 不应依赖跨区域拖拽完成核心流程。
- 未配置云存储时，容量提示保持可见但不能挤占核心素材列表。

### 7.5 移动端行为

移动端不做左右分屏，原因是屏幕宽度不足，分屏会让 iframe 和生成表单都不可用。

移动端要求：

- 素材灵感库和资源物料仓库继续使用全屏 Drawer。
- Drawer 内强调“使用素材后返回工作台”的短路径。
- 资源物料使用后可以自动关闭 Drawer 或展示“继续选择 / 返回工作台”二选一，默认建议返回工作台。
- 灵感网站仍支持外部打开，因为移动端 iframe 可用空间和第三方登录体验都更差。
- 移动端全屏 Drawer 必须尊重 safe-area 和软键盘，不遮挡底部操作。

### 7.6 平板与窄桌面行为

建议使用断点和实际容器宽度共同决定：

| 宽度条件 | 行为 |
| --- | --- |
| `< 900px` | 强制全屏 Drawer |
| `900px - 1199px` | 默认 Drawer，可允许用户手动开启分屏 |
| `>= 1200px` | 桌面默认分屏 |

如果用户手动将桌面窗口缩窄，已打开的分屏面板应自动转为 Drawer 或折叠为底部/全屏模式，并保留当前站点、筛选和选中素材状态。

## 8. 组件化与模块拆分

### 8.1 新增通用组件建议

建议新增以下通用组件，而不是在素材业务组件中直接改布局：

```text
src/components/ui/resizable-workspace.tsx
src/components/ui/workspace-pane.tsx
src/components/ui/workspace-resize-handle.tsx
src/components/creative-resource-workspace-panel.tsx
src/lib/workspace-panel-preferences.ts
src/types/workspace-panel.ts
```

职责划分：

| 模块 | 职责 |
| --- | --- |
| `ResizableWorkspace` | 负责主区域 + 辅助区域的布局、响应式切换、grid/flex track、尺寸约束和状态分发 |
| `WorkspacePane` | 负责非 Modal 面板外壳、标题栏、工具栏、收起/关闭和可访问性 landmark |
| `WorkspaceResizeHandle` | 负责 pointer/keyboard 调整大小、拖动热区、iframe 拖动遮罩和双击恢复默认 |
| `CreativeResourceWorkspacePanel` | 负责把资源物料仓库和素材灵感库渲染到分屏面板 |
| `AssetLibraryDrawer` | 保留为 Drawer 包装器，复用同一份素材内容组件 |
| `workspace-panel-preferences` | 负责保存和规范化 pane 尺寸、折叠状态、停靠方向、默认打开方式和上次 Tab |

### 8.2 内容组件复用

当前素材库内容不应只绑定 Drawer。建议拆成：

```text
AssetLibrarySurface
├── AssetsPanelContent
├── InspirationPanelContent
├── InspirationSiteManager
└── InspirationIframeViewer
```

然后由不同外壳复用：

```text
AssetLibraryDrawer
└── AssetLibrarySurface variant="drawer"

CreativeResourceWorkspacePanel
└── AssetLibrarySurface variant="split"
```

内容组件需要通过 `variant` 或 capability flags 调整密度：

- `drawer`：适合独立浏览和管理，二级页面可以全屏。
- `split`：首屏更紧凑，隐藏低频维护，强调搜索、列表、站点打开和发送到编辑。
- `mobile`：按钮更大，列表单列，操作更偏向“选择后返回工作台”。

## 9. 全局浮动菜单集成

全局浮动菜单应继续作为素材入口，但需要支持打开方式：

```text
创作资源
├── 素材灵感库
│   ├── 分屏打开
│   ├── 抽屉打开
│   └── 外部打开常用站点
└── 资源物料仓库
    ├── 分屏打开
    └── 抽屉打开
```

交互建议：

- 点击一级功能时使用默认打开方式。桌面端素材灵感库默认分屏，移动端默认 Drawer。
- 菜单项右侧可提供“更多打开方式”二级菜单，避免主菜单变复杂。
- 用户在设置中修改默认打开方式后，菜单主点击行为随之更新。
- 如果当前视口不支持分屏，点击“分屏打开”应自动降级为全屏 Drawer，并给出轻量 Notice。
- 后续新增功能也可以声明 `supportedSurfaces`，由菜单统一渲染，不改主页面布局。

## 10. 状态与偏好保存

建议保存以下用户偏好：

| 偏好 | 默认值 | 保存位置 |
| --- | --- | --- |
| 素材功能桌面默认打开方式 | `split` | 本地配置或 localStorage |
| 素材功能移动默认打开方式 | `drawer` | 本地配置或 localStorage |
| 分屏停靠方向 | `right` | 本地配置或 localStorage |
| 分屏宽度 | `520px` 或上次宽度 | 本地配置或 localStorage |
| 分屏宽度比例 | 按当前宽度换算 | 本地配置或 localStorage |
| 分屏是否收起 | `false` | 本地配置或 localStorage |
| 折叠前宽度 | 上次有效宽度 | 本地配置或 localStorage |
| 上次打开 Tab | `assets` 或上次值 | 本地配置或 localStorage |

兼容要求：

- 所有持久化值必须 normalize，非法值回到安全默认。
- 窗口变窄后，旧宽度不能导致主工作台不可见。
- 用户清理本地配置后，分屏回到右侧默认宽度。
- 拖动结束后才写入持久化，避免拖动时高频写入。
- 布局偏好 P0 只本机保存，不随云同步跨设备同步。
- 这些偏好不包含 API Key、云存储 Secret 或第三方站点 Cookie。

## 11. 可访问性、主题与视觉要求

- 分屏面板使用 `role="complementary"` 或明确的 landmark，不使用 `aria-modal="true"`。
- 面板打开后不应把主工作台设为 inert，也不应加遮罩。
- 面板内的 iframe 必须有可读 `title`。
- 宽度调整手柄支持键盘操作，例如方向键每次调整 24px，`Home`/`End` 调整到最小/最大宽度。
- 宽度调整手柄应有 `role="separator"`、`aria-orientation="vertical"`、`aria-valuemin`、`aria-valuemax` 和 `aria-valuenow`。
- 鼠标拖动、触摸拖动和键盘调整都要可用；不能只依赖 hover。
- `Esc` 只在焦点位于面板内时关闭或返回上一层，不应打断用户在主工作台输入。
- 浅色和深色主题都使用现有 CSS 变量，不硬编码大面积白色/黑色。
- 分屏状态下浮动菜单不能遮挡面板关闭按钮、生成按钮或源图操作区。
- 所有新增可见文案进入 `src/lib/i18n/messages.ts` 并覆盖所有支持语言。

## 12. 跨端实现要求

### 12.1 Web

- 分屏布局纯前端实现，不依赖 Next API Route，保证静态导出路径可用。
- iframe 仍是最佳努力能力；不能代理第三方页面。
- Clipboard API 读取图片必须由用户点击触发，并处理权限拒绝。
- 如果浏览器不支持读取图片剪贴板，降级为源图区域接收粘贴事件。
- 拖动调整尺寸使用 Pointer Events，避免只支持鼠标事件。

### 12.2 Tauri 桌面

- 外部打开继续走 `openExternalUrl`。
- 系统剪贴板读取和本地文件操作通过 `src/lib/desktop-runtime.ts` 封装，不在业务组件直接导入 Tauri API。
- 分屏布局仍在 WebView 内完成，不需要创建原生多窗口作为 P0。
- 如果未来增加原生多窗口或独立素材窗口，应作为 P2 能力，不替代分屏 P0。

### 12.3 Tauri Android / 移动端

- 默认不启用分屏。
- 全屏 Drawer 使用现有可复用 Drawer，并保持移动端可用空间。
- 读取剪贴板和文件选择走移动端允许的权限路径；失败时提供手动上传/粘贴降级。

## 13. 实施阶段建议

### P0 - 解决阻断问题

- 新增 `ResizableWorkspace`、`WorkspacePane` 和 `WorkspaceResizeHandle` 通用组件。
- 实现类似 VS Code 的左右拖动调整大小能力，包括 pointer 拖动、键盘调整、双击恢复默认、最小/最大宽度约束和拖动结束持久化。
- 将素材库内容从 Drawer 外壳中抽离，使 Drawer 和分屏面板复用同一业务内容。
- 在 `src/app/page.tsx` 中接入分屏状态，让主工作台和素材面板并排显示。
- 全局浮动菜单支持桌面端默认分屏打开素材灵感库和资源物料仓库。
- 分屏面板支持关闭、收起、右侧停靠、折叠恢复和宽度限制。
- 素材灵感库 iframe 在分屏面板内打开；失败时外部打开。
- 资源物料仓库在分屏面板中发送素材到编辑区后默认不关闭。
- 主工作台提供明确的“从剪贴板添加图片”入口或等价粘贴路径。
- 完成 i18n、浅色/深色、桌面/移动基本验证。

### P1 - 提升效率与可配置性

- 支持左侧/右侧停靠偏好。
- 支持桌面快捷键打开/关闭分屏面板。
- 支持面板内部站点历史、后退/前进、刷新和常用站点快捷入口。
- 支持资源物料多选批量发送到编辑区。
- 支持分屏和 Drawer 之间无损切换，保留当前搜索、分类、选中素材和 iframe URL。
- 支持底部 activity pane 的布局预留和内部 API，但不一定开放 UI。

### P2 - 高级工作区

- 支持多个可注册的工作区面板类型，例如批量中心、同步活动、提示词模板。
- 支持更完整的 VS Code 式 activity bar + panel tabs + bottom panel 组合。
- 支持用户自定义默认功能、面板排序和工作区布局预设。
- 评估 Tauri 桌面原生独立窗口作为高级选项，但不能取代应用内分屏。

## 14. 关键风险与调整

| 风险 / 矛盾 | 调整方案 |
| --- | --- |
| Drawer 覆盖主工作台，无法边参考边生成 | 桌面端连续参考场景默认使用非 Modal 分屏面板 |
| 分屏压缩主工作台导致布局拥挤 | 主工作台按可用宽度自动从两列降为单列，面板有最小/最大宽度 |
| 固定宽度侧栏无法适配不同屏幕和用户习惯 | P0 支持左右拖动调整、双击恢复默认、折叠和持久化 |
| 后续更多工作区块加入后再次重构布局 | P0 就抽象 pane/layout/resize handle，不把素材面板写死在布局层 |
| iframe 抢占 pointer 导致拖动分隔线不稳定 | 拖动时显示透明覆盖层捕获事件，拖动结束移除 |
| iframe 内复制后父页面无法自动接收粘贴 | 提供主工作台粘贴入口和剪贴板读取按钮，不承诺后台自动读取 |
| 第三方站点拒绝 iframe | 展示失败态并外部打开，不代理绕过 |
| 功能菜单打开方式过多变复杂 | 主点击用默认打开方式，更多打开方式放二级菜单 |
| 分屏组件被素材业务绑死 | 抽出通用 `ResizableWorkspace` / `WorkspacePane` / `WorkspaceResizeHandle`，素材只是首个使用方 |
| 移动端分屏不可用 | 移动端保持全屏 Drawer，并优化返回工作台路径 |

## 15. 验收标准

1. 桌面端从全局浮动菜单打开素材灵感库时，默认进入分屏布局，主工作台仍可操作。
2. 桌面端分屏状态下，用户可以在右侧浏览灵感网站，同时在左侧粘贴源图、输入提示词并发起生成。
3. 桌面端分屏状态下，用户从资源物料仓库发送素材到编辑区后，素材面板保持打开。
4. 用户可以关闭或收起分屏面板，关闭后主工作台状态不丢失。
5. 用户可以拖动主工作台和右侧辅助面板之间的分隔线，实时调整两侧宽度。
6. 分隔线支持键盘调整和双击恢复默认宽度，拖动结束后宽度被持久化。
7. 分屏面板宽度受到边界约束，不会让主工作台不可见或产生页面级横向滚动。
8. 面板内有 iframe 时，拖动分隔线仍稳定可用，不被 iframe 截断 pointer 事件。
9. 移动端打开素材功能仍为全屏 Drawer，不出现不可用的窄分屏。
10. iframe 被拒绝或超时时，右侧面板显示清晰失败态，并提供外部打开入口。
11. Drawer 仍可用于素材管理、网站管理、导入导出等低频任务。
12. 分区布局组件是通用组件，不把素材业务逻辑写死在布局组件内。
13. 所有新增可见文案完成 i18n，浅色/深色主题均可读。

## 16. 建议测试范围

- 单元测试：
  - 分屏状态 normalize、宽度 clamp、断点降级。
  - pane/layout 状态恢复、非法值归一化、折叠前宽度恢复。
  - resize handle 的 pointer delta、keyboard delta、双击 reset 和最小/最大值约束。
  - 功能菜单 `supportedSurfaces`、默认打开方式和降级逻辑。
  - 素材内容组件在 `drawer` / `split` variant 下的关键渲染差异。
- 集成测试：
  - 从浮动菜单打开分屏面板，再发送资源物料到编辑区。
  - 拖动分隔线改变右侧面板宽度，刷新后恢复合理宽度。
  - 拖动到窄屏后自动 clamp 或降级，不产生横向滚动。
  - 分屏与 Drawer 切换后保留素材搜索、分类和当前 Tab。
  - iframe 失败态外部打开路径。
  - 剪贴板读取失败时的降级提示。
- UI/E2E：
  - 桌面宽屏分屏：左侧工作台可输入、可粘贴、可生成。
  - 桌面宽屏拖拽：分隔线可拖动，拖动时 iframe 不抢事件，主工作台和面板都不重置。
  - 桌面窄屏降级：分屏自动变 Drawer 或折叠，不横向溢出。
  - 移动端全屏 Drawer：安全区、软键盘、关闭和返回工作台路径。
  - 浅色/深色主题下，分屏边界、工具栏、iframe 失败态和浮动菜单都可读。
