# 分享与云同步

分享和同步解决的是两个不同问题：

- **分享链接**：把当前提示词和可选配置交给别人或自己的另一个浏览器。
- **云同步**：把配置、提示词、历史记录和历史图片同步到 S3 兼容对象存储，再从其他设备恢复。

## 分享当前配置

点击提示词工具栏里的 **分享** 可以打开分享面板。

![分享链接](./images/share-dialog.png)

可以选择写入链接的内容：

- 当前提示词。
- 当前模型 ID。
- 当前命名供应商端点。
- API Base URL。
- 自动生成开关。
- API Key。
- 云存储同步配置和恢复策略。

默认推荐分享提示词、模型和端点，不推荐分享 API Key。

## 普通分享

普通分享会把参数写入 URL。接收者打开后，应用会自动填入这些参数，然后清理地址栏里已经消费的参数，减少误分享敏感信息的风险。

适合：

- 分享一个提示词模板给同事。
- 把某个模型和端点组合交给另一个浏览器使用。
- 做自动化入口，打开后自动生成。

不适合：

- 明文分享长期有效 API Key。
- 分享私有内网 Base URL 给外部用户。

## 密码加密分享

启用 **使用密码加密整个分享链接** 后，链接只暴露 `sdata` 参数，提示词、模型和配置会整体加密。

输入解密密码并在确认框中填入完全一致的密码后，分享面板会自动生成加密后的长链接，不需要先点击复制。密码为空或两次不一致时，输入框会暂时显示干净的应用入口链接；待密码一致后会实时替换为带 `sdata` 的加密链接。点击 **创建短链** 时也会使用这个当前加密链接。

密码有两种传递方式：

- 不写入链接：通过另一条消息或可信渠道发送密码。
- 写入 `#key=`：复制出来的完整链接可以自动解密，但拿到完整链接的人也等同拿到了密码。

如果链接包含 API Key 或云存储访问凭据，建议一定使用密码加密，并只发给可信接收者。

## 分享短链

如果部署管理员在后台开启了短链功能，分享面板会显示 **创建短链** 区域。

短链只会在你明确点击 **创建短链** 时创建。打开分享面板、修改分享选项、复制普通链接、生成加密链接或链接长度超过提示阈值，都不会自动创建短链，避免产生大量无用记录。

短链创建规则由后台控制：

- 管理员可以关闭短链功能。
- 管理员可以要求输入创建口令。
- 明文 API Key 或云存储同步配置默认不允许存入短链；应先使用密码加密分享。
- 加密分享如果勾选了 `#key=` 自带解密密码，也可以创建短链；短链会保存创建当刻的完整目标 URL，因此拿到短链数据库目标 URL 的人也等同拿到了解密密码。

创建成功后，短链会指向创建当刻的分享链接快照。之后继续修改分享选项不会自动更新旧短链，需要再次点击创建新的短链。

同一个部署如果绑定了多个域名，短链地址会使用创建请求的实际域名。例如在 `i.anzz.site` 打开分享面板创建短链时，返回的是 `https://i.anzz.site/s/{code}`，不会强制改成配置里的回退域名。

### Web 端 DevTools 抑制

Web 部署默认会在分享入口启用 DevTools 抑制，给临时分享链接增加一层额外摩擦。可以用环境变量调整作用域：

```dotenv
NEXT_PUBLIC_DISABLE_DEVTOOL_SCOPE=share
```

它有三个值：

- `none`：不启用。
- `all`：整个 Web 应用启用。
- `share`：只在分享入口页启用，默认值，适合这个场景。

这个功能只是在浏览器里增加一点门槛，减少“打开开发者工具就一眼看到 key”的情况，不是安全边界。分享出去的 API Key 仍然应该是临时的、限额的、可控的。

因为这是 `NEXT_PUBLIC_*` 配置，Web 端改完后需要重新构建或重启部署才会生效。未配置时使用 `share`。

## 接收分享链接时的选择

如果链接包含 API 配置，接收者会看到选择：

- **仅本次使用**：刷新后不保留。
- **保存到本地设置**：写入当前浏览器，以后继续使用。
- **忽略这些配置**：不使用分享来的 API 配置。

保存包含 API Base URL 和 API Key 的分享时，应用会按供应商端点合并配置：Base URL 已存在则更新该端点下的 API Key；Base URL 不同则新增一个供应商端点，并默认选中这次分享带来的端点。这样多个不同中转端点或不同账号可以并存，不会互相覆盖默认配置。

共享电脑或不可信链接建议选择“仅本次使用”或“忽略”。

分享 API Key 时选择 **只允许临时使用** 后，分享面板会默认勾选 **禁止批量能力**。接收者打开这类链接后，批量入口会保持可见但不可用，避免临时 Key 的额度被一次性批量消耗；分享者可以在创建链接前手动取消这个限制。

## 云存储同步

云同步在 **Settings -> 云存储同步** 中配置。

![云同步设置](./images/settings-cloud-sync.png)

支持 S3 兼容对象存储，例如：

- AWS S3。
- Cloudflare R2。
- MinIO。
- RustFS。
- Backblaze B2 S3。

同步内容包括：

- 应用配置。
- 提示词历史。
- 用户提示词模板。
- 生成历史元数据。
- 创作工作空间元数据和历史归属。
- IndexedDB 历史图片。

图像供应商 API Key 在快照中会脱敏；S3 同步配置本身也不会写入同步快照。

工作空间只同步内容归属，不会为不同工作空间保存独立的供应商、模型、比例、运行参数或云存储配置。恢复快照时，历史会保留原始 `workspaceId`；旧快照没有工作空间归属时才会进入默认工作空间。删除工作空间会写入本地 tombstone，避免后续恢复把已删除空间自动混回当前历史。

### 对象存储权限

默认同步不需要删除权限。建议给同步凭据授予读取、列出和写入当前前缀的权限，例如 `GetObject`、`ListBucket`、`PutObject`；不要默认授予 `DeleteObject`。

如果本地删除了历史图片而远端仍有旧图片，普通同步会继续上传新文件并保留旧远端图片引用，不会发布删除标记，也不会请求删除远端对象。只有在 **云存储同步** 设置里显式开启 **允许同步删除远端图片** 后，本地删除才会同步为远端删除标记，并且“同时删除远端图片”选项才会出现。

## 同步与恢复的区别

历史区右上角的云图标会提供手动操作：

- **自动同步**：可直接在云图标菜单里开启或关闭，和系统设置中的自动同步开关保持一致。
- **同步配置**：上传轻量配置和历史记录。
- **同步历史图片**：上传历史图片 Blob。
- **同步最近图片**：只上传最近一段时间内的图片。
- **强制同步历史图片**：重新检查并上传。
- **恢复配置**：下载并应用配置、提示词和历史元数据。
- **恢复历史图片**：下载并校验图片文件。
- **恢复最近图片**：只恢复最近一段时间内的图片。
- **强制恢复历史图片**：忽略本地已有状态重新恢复。

配置和图片可以分开同步，避免每次都传输大量图片。

多选生成历史或图生文历史后，工具条会提供 **同步** 按钮，只上传选中条目关联的图片或源图，适合补同步少量漏传内容。

即使尚未配置云存储，历史面板也默认显示云同步入口、单条上传按钮和同步状态图标。点击相关操作时会提示先在系统设置中完成 S3 兼容对象存储配置。若某个设备不需要云存储，可在 **系统设置 → 云存储同步** 中开启 **未配置时隐藏云存储入口**；一旦云存储配置完整，该隐藏选项会置灰，同步入口始终显示。

## 自动同步

云同步设置中可以开启自动同步，也可以从生成历史右上角的云图标菜单临时开启或关闭。自动同步的内容范围仍由设置页控制：

- 应用配置。
- 润色提示词。
- 提示词历史。
- 提示词模板。
- 图片历史。
- 历史图片。

建议先手动验证一次上传和恢复，再开启自动同步。

## CORS 与桌面端

Web 端默认使用浏览器直连对象存储，因此对象存储需要放行当前站点的 CORS 请求。

如果遇到 CORS 问题：

- 检查 Bucket CORS 是否允许 `GET`、`PUT`、`HEAD`、`OPTIONS`。
- 暴露 `ETag` 和 `x-amz-meta-sha256`。
- 使用桌面端绕开浏览器 CORS 限制。
- 在可信自部署环境中切换为服务器中转。
