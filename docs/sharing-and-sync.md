# 分享与云同步

分享和同步解决的是两个不同问题：

- **分享链接**：把当前提示词和可选配置交给别人或自己的另一个浏览器。
- **云同步**：把配置、提示词、历史记录和历史图片同步到 S3 兼容对象存储，再从其他设备恢复。

## 分享当前配置

点击提示词工具栏里的 **分享** 可以打开分享面板。

![分享链接](./images/share-dialog.png)

可以选择写入链接的内容：

- 当前提示词。
- 当前模型 ID。
- 当前命名供应商端点。
- API Base URL。
- 自动生成开关。
- API Key。
- 云存储同步配置和恢复策略。

默认推荐分享提示词、模型和端点，不推荐分享 API Key。

## 普通分享

普通分享会把参数写入 URL。接收者打开后，应用会自动填入这些参数，然后清理地址栏里已经消费的参数，减少误分享敏感信息的风险。

适合：

- 分享一个提示词模板给同事。
- 把某个模型和端点组合交给另一个浏览器使用。
- 做自动化入口，打开后自动生成。

不适合：

- 明文分享长期有效 API Key。
- 分享私有内网 Base URL 给外部用户。

## 密码加密分享

启用 **使用密码加密整个分享链接** 后，链接只暴露 `sdata` 参数，提示词、模型和配置会整体加密。

密码有两种传递方式：

- 不写入链接：通过另一条消息或可信渠道发送密码。
- 写入 `#key=`：复制出来的完整链接可以自动解密，但拿到完整链接的人也等同拿到了密码。

如果链接包含 API Key 或云存储访问凭据，建议一定使用密码加密，并只发给可信接收者。

## 接收分享链接时的选择

如果链接包含 API 配置，接收者会看到选择：

- **仅本次使用**：刷新后不保留。
- **保存到本地设置**：写入当前浏览器，以后继续使用。
- **忽略这些配置**：不使用分享来的 API 配置。

共享电脑或不可信链接建议选择“仅本次使用”或“忽略”。

## 云存储同步

云同步在 **Settings -> 云存储同步** 中配置。

![云同步设置](./images/settings-cloud-sync.png)

支持 S3 兼容对象存储，例如：

- AWS S3。
- Cloudflare R2。
- MinIO。
- RustFS。
- Backblaze B2 S3。

同步内容包括：

- 应用配置。
- 提示词历史。
- 用户提示词模板。
- 生成历史元数据。
- IndexedDB 历史图片。

图像供应商 API Key 在快照中会脱敏；S3 同步配置本身也不会写入同步快照。

## 同步与恢复的区别

历史区右上角的云图标会提供手动操作：

- **同步配置**：上传轻量配置和历史记录。
- **同步历史图片**：上传历史图片 Blob。
- **同步最近图片**：只上传最近一段时间内的图片。
- **强制同步历史图片**：重新检查并上传。
- **恢复配置**：下载并应用配置、提示词和历史元数据。
- **恢复历史图片**：下载并校验图片文件。
- **恢复最近图片**：只恢复最近一段时间内的图片。
- **强制恢复历史图片**：忽略本地已有状态重新恢复。

配置和图片可以分开同步，避免每次都传输大量图片。

## 自动同步

云同步设置中可以开启自动同步，并选择同步范围：

- 应用配置。
- 润色提示词。
- 提示词历史。
- 提示词模板。
- 图片历史。
- 历史图片。

建议先手动验证一次上传和恢复，再开启自动同步。

## CORS 与桌面端

Web 端默认使用浏览器直连对象存储，因此对象存储需要放行当前站点的 CORS 请求。

如果遇到 CORS 问题：

- 检查 Bucket CORS 是否允许 `GET`、`PUT`、`HEAD`、`OPTIONS`。
- 暴露 `ETag` 和 `x-amz-meta-sha256`。
- 使用桌面端绕开浏览器 CORS 限制。
- 在可信自部署环境中切换为服务器中转。
