# 生成与编辑图片

GPT Image Playground 的核心工作流有两类：从文字生成图片，以及基于源图片继续编辑。两类工作流使用同一个提示词框和高级选项，只是是否添加源图片不同。

## 文生图

适合从零开始生成素材，例如商品主图、社媒封面、活动 KV、头像、场景概念图。

基本步骤：

1. 在提示词框描述画面。
2. 点击 **高级** 选择供应商和模型。默认模型会来自系统设置里的 **任务默认**。
3. 设置图片数量、尺寸、质量和格式。
4. 点击 **开始生成**。
5. 在输出区查看结果，并把满意的图片下载或发送到编辑。

![高级选项](./images/advanced-options.png)

## 图片数量

图片数量可以设为 1-10 张。多图适合探索方向，单图适合最终输出。

流式预览只在支持的模型和单图生成时可用。数量大于 1 时，系统会自动关闭流式预览。

## 尺寸与质量

常见尺寸入口：

- **自动**：交给模型判断尺寸或比例。
- **纵向**：适合短视频封面、海报、手机壁纸。
- **横向**：适合官网首屏、横幅、发布会主视觉。
- **正方形**：适合头像、商品图、社交平台图片。
- **自定义**：在支持的模型上输入宽高。

高级尺寸区还提供 **按场景选择尺寸**。它会打开独立弹层，按社交媒体、广告、网页、电商、印刷、屏幕、品牌资产等场景筛选比例和分辨率。选择后会把当前模型可提交的尺寸回填到尺寸控件下方，并继续提供清晰度、当前比例、对调比例和同一比例下分辨率切换。

场景尺寸会区分原始平台推荐尺寸和模型实际提交尺寸。例如 `gpt-image-2` 会转换为符合 16 的倍数、最长边、总像素和 `1:3` 到 `3:1` 比例限制的 `WIDTHxHEIGHT`；Gemini 等枚举尺寸模型会映射到最接近的官方尺寸档位。刷新页面后，如果当前模型和尺寸仍匹配，场景回填状态会继续保留；手动改用原有尺寸控件或切换到不兼容模型时会清除场景标记。

`gpt-image-2` 支持自定义宽高。界面会校验边长、总像素、宽高比和 16 的倍数，避免提交无效尺寸。

自定义推荐会给出多个可提交候选，尽量覆盖更小、接近和更大的清晰度档位。OpenAI 当前要求长短边比例不超过 `3:1`，因此 `1200x300` 这类 `4:1` 输入无法生成同样 `4:1` 的合规候选；界面会明确提示已按最接近的 `3:1` 上限推荐，并从满足总像素下限的尺寸开始。

## 输出格式

根据用途选择：

- **PNG**：适合保留透明背景或后期处理。
- **JPEG**：适合普通预览和较小体积。
- **WebP**：适合 Web 页面和批量素材。

当 JPEG 或 WebP 支持压缩控制时，会出现压缩率滑块。

## 批量规划

提示词工具栏里的 **批量** 可以把当前提示词和源图片上下文规划成多个可编辑任务。主提示词为空时也可以打开批量面板，先添加待处理图片或导入任务；有提示词时按钮会保持高亮，并把当前提示词带入批量流程。

批量面板支持 AI 规划、批量改图、文本切分和 JSON 导入；AI 规划需要先在 **Settings -> 批量配置** 中选择批量规划模型，批量改图、文本切分和 JSON 导入不需要模型。

**批量改图** 有独立的待处理图片区，支持多选图片、拖拽图片，以及在支持的浏览器或桌面 WebView 中拖入/选择文件夹。这里的图片会逐张生成图片编辑任务，不会写入主表单的源图片区；主源图片区的图片会作为整批共享参考图附加到每条改图任务。确认入队后，任务和历史会保留原始文件名、相对路径、输入图序号和变体序号，方便对应回原图。

选择“风格迁移”“老照片修复”“图片增强”或“背景重绘”等批量改图模式时，系统会先把对应的预设指令写入 **共享改图指令**，你可以在生成预览前直接改成更适合本批图片的描述。没有共享参考图时，预览任务会直接使用共享改图指令，不额外插入文件名或图片序号说明。

批量配置还可以设置默认规划方式、AI 自动规划提示词、单次预览最大条目数和确认前提醒阈值。生成预览后，右侧结果区会显示可逐条编辑、禁用、重排或重新生成的任务列表；只有点击确认后，任务才会进入现有队列。默认启用任务数超过 100 时，系统会先弹出二次确认，避免误提交过大的批次。

## 参考图编辑

添加源图片后，输入区会自动切换为编辑图片模式。

![源图片编辑](./images/edit-source-image.png)

适合这些场景：

- 给人物或商品添加元素。
- 改变图片风格。
- 替换背景或光影。
- 保留主体，调整构图和氛围。
- 多张参考图合成一个画面。

源图片数量、格式和大小会按当前模型动态限制，而不是固定 10 张。`gpt-image-2` 支持最多 16 张 PNG/JPEG/WebP 参考图，单图上限 50MB；Seedream 支持最多 14 张 JPEG/PNG/WebP/BMP/TIFF/GIF/HEIC/HEIF 参考图，单图按 30MB 限制，并校验 36MP、最短边和宽高比；`doubao-seedream-5.0-lite` 还会按“参考图数量 + 输出图数量 <= 15”动态收紧；Gemini Nano Banana 2 支持最多 14 张 PNG/JPEG/WebP/HEIC/HEIF 参考图，当前 inline 传输按总 20MB 控制。可以使用文件选择器，也可以拖拽或粘贴图片。添加多张后，可以直接拖动缩略图调整顺序；提交编辑、图生文、图生视频或批量任务时会按调整后的顺序发送。

## 蒙版编辑

在支持蒙版的模型上，添加源图片后可以在高级选项里创建蒙版。

![蒙版编辑器](./images/mask-editor.png)

使用方式：

1. 添加源图片。
2. 点击 **高级 -> 蒙版 -> 创建蒙版**。
3. 在图片上绘制需要编辑的区域。
4. 调整笔刷大小。
5. 点击 **保存蒙版**。
6. 提交编辑任务。

你也可以上传已有 PNG 蒙版。上传的蒙版尺寸必须与源图片一致。

## Seedream 和 SenseNova 高级参数

不同供应商有自己的参数。应用会优先把常用参数做成界面控件，减少手写 JSON。

![Seedream 高级参数](./images/seedream-options.png)

Seedream 常见参数包括：

- 尺寸和 2K/3K/4K 预设。
- 响应格式：URL 或 Base64 JSON。
- 水印。
- 序列生成和最大图片数。
- Seed、Guidance Scale。
- 输出格式。
- 提示词优化模式。
- 联网搜索。

SenseNova 会展示它支持的固定尺寸列表，例如 1:1、16:9、9:16、3:2、21:9 等。

## 自定义参数 JSON

高级选项底部有 **自定义参数 JSON**。它适合临时使用供应商新增、低频或尚未做成控件的参数。

建议：

- 常用参数优先使用界面控件。
- JSON 只作为临时兜底。
- 同名字段会覆盖界面生成的参数。
- 提交前界面会检查 JSON 是否有效。

## 继续编辑已有结果

你可以从三个地方把图片继续发送到编辑：

- 输出区的 **编辑** 按钮。
- 历史卡片预览。
- 全屏预览里的发送到编辑入口。

发送后页面会回到编辑区，并自动加载这张图片作为源图。
