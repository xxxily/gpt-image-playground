# 广告与后台管理使用手册

这份文档对应当前代码里的广告位、广告组、分享 Profile 和桌面读取实现。广告内容只能由管理员在后台维护，普通用户不能通过权限 Key 自行提交广告。

## 先认清三个概念

- **广告位**：页面上的展示位置，例如生成区头部、页面顶部、历史面板上方。
- **广告组**：某个广告位上的一套投放配置，包含名称、备注、启停、时间窗、轮播间隔和切换方式。
- **轮播素材**：广告组里的单张图片。多张素材放在同一个广告组里，前端会按排序轮播。

展示优先级是：

1. 分享广告组：分享链接带 `promoProfileId`，且后台存在有效分享广告组。
2. 全局广告组：普通访问或分享广告失效时使用。
3. 旧环境变量广告：仅作为 `generation_form_header` 的兼容兜底。

## 管理员怎么配置

管理员入口是 `/admin`，广告入口是 `/admin/promo`。

### 1. 广告位

在 **广告位** 区域确认展示位置是否存在。当前默认广告位有：

- `generation_form_header`
- `app_top_banner`
- `history_top_banner`

广告位控制位置本身是否可用，以及默认轮播间隔和默认切换方式。

### 2. 全局广告组

在 **广告组 -> 全局广告组** Tab 中创建。

全局广告组需要填写：

- 名称：后台识别用。
- 备注：投放目的、客户、素材来源或审核说明。
- 广告位：选择展示位置。
- 启用状态。
- 开始/结束时间：留空表示立即开始或长期有效。
- 间隔和切换：留空间隔时继承广告位默认值。

### 3. 分享广告组

在 **广告组 -> 分享广告组** Tab 中创建。

分享广告组同样有名称、备注、广告位、启停、时间窗、间隔和切换方式。不同点是：保存后系统会自动生成 `Profile ID`，管理员只需要复制这个 ID 给用户。

管理员不需要、也不能手动填写 `Profile ID`。这样可以避免外部用户绕过审核自行提交广告。

用户拿到 `Profile ID` 后，可以在分享面板的 **广告 Profile ID** 中填写，生成的分享链接会携带：

```text
?promoProfileId=系统生成的ID
```

### 4. 管理轮播素材

不要在广告组列表页直接编辑素材。现在每个广告组都有独立素材页：

1. 进入 **全局广告组** 或 **分享广告组** Tab。
2. 找到目标广告组。
3. 点击 **管理素材**。
4. 在子页面里新增、编辑、停用或删除该广告组的素材。

同一个广告组下可以放多张素材：

- `排序` 小的先显示。
- `排序` 相同时，`权重` 大的优先。
- `设备` 为 `all` 时桌面和移动都展示。

## 分享 Profile 的使用逻辑

分享 Profile 不是权限 Key。它只是一个公开读取 ID。

- 管理员创建分享广告组。
- 系统自动创建 `Profile ID`。
- 管理员审核并配置素材。
- 管理员复制 `Profile ID` 给用户。
- 用户只把这个 ID 放进分享链接。

如果分享广告组被停用、过期、素材全部失效或广告位被停用，前端会降级到全局广告组；没有全局广告时则隐藏广告位。

## 桌面端怎么读广告

在 **Settings -> 桌面端设置 -> 广告读取** 里配置：

- **当前站点**：读取当前站点的 `/api/promo/placements`
- **自定义域名**：填写域名，系统自动拼成广告读取接口
- **完整接口**：直接填完整读取地址
- **关闭**：不请求广告接口

桌面端请求失败、离线或超时后，会直接隐藏广告，不会影响生成、编辑和历史。

## 当前页面上的广告位

- 生成区头部：在编辑卡片标题右侧
- 页面顶部：在主页面标题下方
- 历史面板上方：在历史面板标题前

## 常见问题

### 为什么我只看见一张图？

先检查：

- 广告位是否启用
- 广告组是否启用且在有效时间窗内
- 素材是否至少有 2 张
- 素材是否启用且在有效时间窗内
- 素材的 `sortOrder` 和 `weight` 是否符合预期

### 为什么分享广告没出来？

先检查：

- 分享链接是否带了正确的 `promoProfileId`
- 后台分享广告组是否启用且未过期
- 分享广告组是否绑定了目标广告位
- 轮播素材是否有效
- 目标广告位是否启用

### 为什么一个广告位有多个广告组？

这是允许的。公共读取会选中当前有效且最近更新的那一组。
如果只是想做轮播，应该把多张图放进同一个广告组，而不是拆成多个广告组。
