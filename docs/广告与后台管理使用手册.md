# 广告与后台管理使用手册

这份文档对应当前代码里的广告、权限 Key、分享广告和桌面读取实现。重点是把实际操作路径说清楚，而不是解释代码结构。

## 先认清三个概念

- **广告位**：展示位置，例如生成区头部、页面顶部、历史面板上方。
- **广告组**：一套生效配置。一个广告位可以有多组配置，但同一时刻公共读取只会选中当前有效的那一组。
- **轮播素材**：广告组里的单张图片。多张轮播图放在同一个广告组里，就会在前端轮播展示。

优先级是：

1. 分享广告
2. 全局广告
3. 旧环境变量广告

## 管理员怎么配

管理员入口是 `/admin`。

### 1. 先有广告位

在 **推广位管理** 里先确认广告位是否存在。当前默认广告位有：

- `generation_form_header`
- `app_top_banner`
- `history_top_banner`

### 2. 创建广告组

在 **广告组** 区域新增一条配置：

- `范围` 选 `global` 时，这是全局广告。
- `范围` 选 `share` 时，这是分享广告。
- `间隔` 和 `切换` 控制轮播节奏。
- `开始/结束` 控制时间窗。

### 3. 添加多张轮播素材

同一个广告组下可以放多张素材。

推荐做法：

- 先选定一个广告组。
- 点击该行里的 **新增素材**。
- 继续补第二张、第三张图。
- 用 `排序` 控制前后顺序，用 `权重` 控制同排序下的优先级。

判断规则是：

- `sortOrder` 小的先显示。
- `sortOrder` 相同时，`weight` 大的优先。

### 4. 常见配置建议

- 生成区头部广告建议放 1 - 3 张图，避免内容太多。
- 顶部和历史位适合 1 张主图或少量轮播图。
- 如果想临时停掉某个广告组或某张图，直接用停用按钮，不必删除。

## 第三方权限 Key 怎么接入

当前已经实现，方式是 **权限 Key + 公共接口**。  
第三方不需要后台账号，只要拿到你发给他的权限 Key。

### 管理员侧

1. 在 `/admin/promo-keys` 创建一个权限 Key。
2. 把生成出来的明文 Key 发给第三方。
3. 可在创建时限制允许的广告位。

### 第三方侧

第三方调用公共接口创建或更新自己的分享广告配置：

```http
POST /api/promo/share-profiles
PUT  /api/promo/share-profiles
```

请求里至少要带：

- `shareKey`
- `slotKey`
- `name`
- `items`

可选带：

- `promoProfileId`：想更新同一份分享广告时就传它；不传则新建一份。

示例请求：

```json
{
  "shareKey": "your-share-key",
  "name": "品牌投放主图",
  "slotKey": "generation_form_header",
  "promoProfileId": "brand-q2-campaign",
  "enabled": true,
  "intervalMs": 5000,
  "transition": "fade",
  "items": [
    {
      "title": "第一张图",
      "alt": "第一张图",
      "desktopImageUrl": "https://example.com/a.webp",
      "mobileImageUrl": "https://example.com/a-mobile.webp",
      "linkUrl": "https://example.com",
      "device": "all",
      "sortOrder": 0,
      "weight": 100
    },
    {
      "title": "第二张图",
      "alt": "第二张图",
      "desktopImageUrl": "https://example.com/b.webp",
      "mobileImageUrl": "https://example.com/b-mobile.webp",
      "linkUrl": "https://example.com",
      "device": "all",
      "sortOrder": 1,
      "weight": 100
    }
  ]
}
```

接口会返回一个公开的 `promoProfileId`。  
第三方后续继续更新同一份广告时，直接复用这个 `promoProfileId`。

### 分享链接怎么用

把 `promoProfileId` 放进分享链接里，主工作台就会优先读取这份分享广告。

例如：

```text
http://localhost:3000/?promoProfileId=brand-q2-campaign
```

如果链接还带有加密分享参数，`promoProfileId` 也会作为公开参数一起保留。

## 桌面端怎么读广告

在 **Settings -> 桌面端设置 -> 广告读取** 里配置：

- **当前站点**：读取当前站点的 `/api/promo/placements`
- **自定义域名**：填写域名，系统自动拼成广告读取接口
- **完整接口**：直接填完整读取地址
- **关闭**：不请求广告接口

桌面端请求失败、离线或超时后，会直接隐藏广告，不会影响生成、编辑和历史。

## 当前页面上的广告位

- 生成区头部：在编辑卡片标题右侧
- 页面顶部：在主页面标题下方
- 历史面板上方：在历史面板标题前

## 常见问题

### 为什么我只看见一张图？

先检查这几个点：

- 广告位是否启用
- 广告组是否启用
- 轮播素材是否至少有 2 张
- 素材的 `sortOrder` 和 `weight` 是否符合预期

### 为什么分享广告没出来？

先检查这几个点：

- 权限 Key 是否有效、未停用、未撤销、未过期
- 允许的广告位里是否包含目标 `slotKey`
- 分享配置是否有有效的 `promoProfileId`
- 分享链接里是否带了正确的 `promoProfileId`

### 为什么一个广告位有多个配置？

这是允许的，但公共读取只会取当前有效的那一组。  
如果你只是想做轮播，通常应该把多张图放进同一个广告组里，而不是拆成多个组。
