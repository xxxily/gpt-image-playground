# 展示内容与后台管理使用手册

这份文档对应当前代码里的展示位、展示组、分享 Profile 和桌面读取实现。展示内容只能由管理员在后台维护，普通用户不能通过权限 Key 自行提交内容。

## 先认清三个概念

- **展示位**：页面上的展示位置，例如生成区头部、页面顶部、历史面板上方。
- **展示组**：某个展示位上的一套展示配置，包含名称、备注、启停、时间窗、轮播间隔和切换方式。
- **展示素材**：展示组里的单张图片。多张素材放在同一个展示组里，前端会按排序轮播。

展示优先级是：

1. 分享展示组：分享链接带 `promoProfileId`，且后台存在有效分享展示组。
2. 全局展示组：普通访问或分享展示组失效时使用。
3. 旧环境变量展示内容：仅作为 `generation_form_header` 的兼容兜底。

## 管理员怎么配置

管理员入口是 `/admin`，展示内容入口是 `/admin/promo`。

### 1. 展示位

在展示位区域确认展示位置是否存在。当前默认展示位有：

- `generation_form_header`
- `app_top_banner`
- `history_top_banner`

展示位控制位置本身是否可用，以及默认轮播间隔和默认切换方式。

### 1.1 展示组比例

展示比例现在由 **展示组** 决定。新建全局展示组或分享展示组时，管理员可以选择常见比例，也可以输入自定义比例。前端会按展示组比例保留展示区域，PC 和移动端使用同一个比例；同一个展示组里的所有素材也必须共用这个比例。

前端使用完整展示方式：图片会在展示组比例容器内完整显示，不会为了铺满容器而裁切。素材图片本身比例与展示组比例一致时效果最好；如果不一致，图片仍会完整显示，但可能出现上下或左右留白。多张素材轮播时，暂停/播放按钮显示在展示图右侧，不会遮挡图片内容。

后台内置的常见比例包括：

- 方形：`1:1`
- 竖版卡片：`4:5`、`3:4`、`2:3`、`9:16`
- 横版卡片：`5:4`、`4:3`、`3:2`、`16:9`
- 分享卡片：`1.91:1`、`2:1`
- 横幅：`3:1`、`4:1`、`5:1`、`10:1`
- 超宽：`21:9`、`12:1`

各展示位的新建默认值是：

| 展示位                   | 默认比例 | 说明                                                                                                          |
| ------------------------ | ------------ | ------------------------------------------------------------------------------------------------------------- |
| `generation_form_header` | `4:1`        | 输入/编辑卡片标题右侧小横幅，空间较小。若选择方形或竖版比例，前端会按比例展示，但需要确认不会挤压主表单区域。 |
| `app_top_banner`         | `10:1`       | 页面标题下方全宽横幅，适合横向信息；也可以按活动物料改为 `16:9`、`4:5`、`1:1` 等比例。                       |
| `history_top_banner`     | `10:1`       | 历史面板上方横幅，尺寸策略与页面顶部一致。                                                                    |

选图注意：

- 先根据已有物料选择展示组比例，再上传素材，不需要为了固定展示位比例重做物料。
- 同一展示组内的桌面图和移动图建议使用同一比例，只针对文字大小、信息密度和细节做不同版本。
- 生成区头部空间较小；如果沿用默认 `4:1`，素材内容应保持简洁，避免小字号长句。
- 关键内容放在画面中间，不要贴边；即使完整显示，边缘内容也容易被圆角或浏览器缩放影响。
- 建议导出 WebP/AVIF，单张控制在 `150 KB` 以内；如果是透明 PNG，要确认浅色和深色主题都能看清。
- 如果素材必须包含二维码，优先放在画面中部偏左或中部，并确认在移动端尺寸下仍清晰可扫。

### 2. 全局展示组

在全局展示组 Tab 中创建。

全局展示组需要填写：

- 名称：后台识别用。
- 备注：展示目的、来源或审核说明。
- 展示位：选择展示位置。
- 展示比例：选择常见比例或自定义比例，同一个展示组内所有素材都会按这个比例展示。
- 启用状态。
- 开始/结束时间：留空表示立即开始或长期有效。
- 间隔和切换：留空间隔时继承展示位默认值。

### 3. 分享展示组

在分享展示组 Tab 中创建。

分享展示组同样有名称、备注、展示位、展示比例、启停、时间窗、间隔和切换方式。不同点是：保存后系统会自动生成 `Profile ID`，管理员只需要复制这个 ID 给用户。

管理员不需要、也不能手动填写 `Profile ID`。这样可以避免外部用户绕过审核自行提交展示内容。

用户拿到 `Profile ID` 后，可以在分享面板的 Profile ID 输入项中填写，生成的分享链接会携带：

```text
?promoProfileId=系统生成的ID
```

### 4. 管理展示素材

不要在展示组列表页直接编辑素材。现在每个展示组都有独立素材页：

1. 进入全局展示组或分享展示组 Tab。
2. 找到目标展示组。
3. 点击 **管理素材**。
4. 在子页面里新增、编辑、停用或删除该展示组的素材。

同一个展示组下可以放多张素材：

- `桌面图 URL` 和 `移动图 URL` 至少填写一个；只填一个时，另一端会沿用同一张图。
- `标题`、`替代文本` 和 `点击链接` 都是可选项；不填点击链接时，前端只展示图片，不提供跳转。
- `排序` 小的先显示。
- `排序` 相同时，`权重` 大的优先。
- `设备` 为 `all` 时桌面和移动都展示。
- 所有素材都使用展示组比例预览和展示；不要在同一个展示组里混放不同比例的物料。

## 分享 Profile 的使用逻辑

分享 Profile 不是权限 Key。它只是一个公开读取 ID。

- 管理员创建分享展示组。
- 系统自动创建 `Profile ID`。
- 管理员审核并配置素材。
- 管理员复制 `Profile ID` 给用户。
- 用户只把这个 ID 放进分享链接。

如果分享展示组被停用、过期、素材全部失效或展示位被停用，前端会降级到全局展示组；没有全局展示内容时则隐藏展示位。

## 短链与推荐内容

管理员入口是 `/admin/short-links`。

短链管理用于把较长的分享链接保存为 `/s/{code}` 形式，同时查看访问情况。短链不会由前端自动创建，用户必须在分享面板里点击 **创建短链**。

后台可以配置：

- 是否开启短链功能。
- 创建模式：关闭创建、仅管理员、管理员 + 创建口令、公开创建。
- 创建口令、短码长度、默认有效期、目标 URL 最大长度。
- 是否允许明文敏感参数进入短链目标；`#key=` 解密密码允许进入短链目标，但会作为风险标记记录。
- 允许创建短链的站点域名。当前请求域名会自动允许；多个域名指向同一服务时，短链返回的域名跟随实际访问域名。

每条短链都可以设置推荐内容策略：

- **继承长链接**：保留原分享链接中的 `promoProfileId`。
- **不展示分享推荐**：跳转时移除 `promoProfileId`。
- **绑定指定 Profile**：跳转时覆盖为后台选定的分享 Profile。

因此，同一个分享内容可以通过不同短链关联不同推荐内容；管理员后续调整短链绑定后，短链 URL 不变，下一次访问立即生效。

## 桌面端怎么读取展示内容

在 **Settings -> 桌面端设置** 里配置展示内容读取方式：

- **当前站点**：读取当前站点的 `/api/promo/placements`
- **自定义域名**：填写域名，系统自动拼成读取接口
- **完整接口**：直接填完整读取地址
- **关闭**：不请求展示内容接口

桌面端请求失败、离线或超时后，会直接隐藏展示内容，不会影响生成、编辑和历史。

## 当前页面上的展示位

- 生成区头部：在编辑卡片标题右侧
- 页面顶部：在主页面标题下方
- 历史面板上方：在历史面板标题前

## 常见问题

### 为什么我只看见一张图？

先检查：

- 展示位是否启用
- 展示组是否启用且在有效时间窗内
- 素材是否至少有 2 张
- 素材是否启用且在有效时间窗内
- 素材的 `sortOrder` 和 `weight` 是否符合预期

### 为什么分享展示内容没出来？

先检查：

- 分享链接是否带了正确的 `promoProfileId`
- 后台分享展示组是否启用且未过期
- 分享展示组是否绑定了目标展示位
- 展示素材是否有效
- 目标展示位是否启用

### 为什么一个展示位有多个展示组？

这是允许的。公共读取会选中当前有效且最近更新的那一组。
如果只是想做轮播，应该把多张图放进同一个展示组，而不是拆成多个展示组。
