# 供应商与系统设置

系统设置是把 GPT Image Playground 变成长期工作台的关键。这里可以配置供应商端点、自定义模型、提示词润色、运行参数、存储、云同步和桌面端选项。

![供应商设置](./images/settings-providers.png)

## 供应商 API 配置

应用内置四类供应商：

- OpenAI Compatible。
- Google Gemini。
- Seedream。
- SenseNova。

每类供应商都会有一个默认端点。你也可以新增多个命名端点，例如：

- OpenAI 官方。
- 公司中转。
- 自己的兼容网关。
- 不同账号或不同额度池。

每个端点可以单独保存：

- 名称。
- API Key。
- API Base URL。
- 可用模型列表。
- 自定义模型 ID。
- 是否默认。

在高级选项里选择供应商时，会显示这些命名端点，而不是只显示抽象供应商名称。

## 自定义模型

如果某个兼容端点提供了项目未预置的模型，可以在该供应商端点的 **可用模型** 区域添加模型 ID。

添加后，它会出现在高级选项的模型列表中。你还可以在 **自定义模型能力覆盖** 中调整它的能力：

- 是否支持编辑。
- 是否支持蒙版。
- 是否支持自定义尺寸。
- 是否支持质量、格式、背景、审核、压缩。
- 是否支持流式预览。
- 默认尺寸和尺寸预设。

## 运行与存储

这里控制工作台运行方式：

- **API 连接模式**：服务器中转或客户端直连。
- **最大并发任务数**：控制同时执行多少生成/编辑任务。
- **提示词历史数量**：限制最近提示词保存数量。
- **图片存储模式**：自动、文件系统或 IndexedDB。
- **图片存储路径**：桌面端可填写本机目录。

## 服务器中转与客户端直连

| 模式 | 数据流 | 适合场景 |
| --- | --- | --- |
| 服务器中转 | 浏览器 -> 应用服务器 -> 模型供应商 | 默认推荐，API Key 不直接暴露给浏览器请求 |
| 客户端直连 | 浏览器 -> 模型供应商 | 第三方中转或自托管端点支持 CORS 时使用 |

直连模式要求目标端点支持 CORS。直连时 API Key 会出现在浏览器网络请求中，不适合公共共享机器。

## 提示词润色配置

润色是独立的 Chat Completions 工作流，可以复用 OpenAI 兼容接口，也可以配置专门的润色模型。

可以配置：

- 润色 API Key。
- 润色 API Base URL。
- 润色模型 ID。
- 是否启用思考模式。
- 默认内置预设。
- 自定义润色提示词。
- 润色下拉顺序。

## 密码验证

部署给多人访问时，可以设置 `APP_PASSWORD`。设置后，提交请求前需要输入密码。

注意：客户端直连不经过服务器，因此不会触发 `APP_PASSWORD` 验证。

## 桌面端设置

桌面端基于 Tauri，可以使用 Rust 中转代理处理部分 Web 端容易遇到的限制，例如对象存储 CORS。

桌面设置包括：

- 桌面代理模式。
- 代理地址。
- 调试模式。
- 图片存储路径。

更多安装和打包说明见 [安装、部署与桌面端](./desktop-and-deployment.md)。
