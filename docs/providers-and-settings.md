# 供应商与系统设置

系统设置是把 GPT Image Playground 变成长期工作台的关键。这里可以配置供应商端点、模型能力、任务默认、提示词润色行为、运行参数、存储、云同步和桌面端选项。

![供应商设置](./images/settings-providers.png)

## 供应商与模型

**供应商与模型** 是 API Key、Base URL、模型读取、模型纳管和能力标注的主入口。它先按职责分成四个入口：

- **供应商端点管理**：统一新增和维护文本、图片、视频供应商端点、API Key、Base URL 和模型白名单。
- **图片模型端点管理**：按图片供应商查看和维护图片生成/编辑端点，新增端点会跳转到统一端点管理。
- **视频端点管理**：按视频供应商模板查看和维护视频端点，新增端点会跳转到统一端点管理。
- **模型能力检视**：跨端点检查模型来源、启用状态和能力标注。

图生文、提示词润色和视频设置页只选择“任务如何消费这些模型”，不再作为日常凭证入口。

**供应商端点管理** 是连接主数据入口。你可以在这里选择文本、图片或视频模板，填写端点名称、API Key 和 API Base URL，然后点击 **获取模型列表** 在弹窗里选择要管理的模型。OpenAI、OpenAI 兼容、Anthropic 或 Anthropic 兼容端点都可以被图生文、提示词润色配置和批量配置复用；对应配置会在用户点击模型状态框时直接读取该端点的最新模型列表，由用户单选绑定到图生文、润色或批量规划任务，避免重复填写同一套供应商信息。

**图片模型端点管理** 内置四类图片兼容供应商：

- OpenAI Compatible。
- Google Gemini。
- Seedream。
- SenseNova。

每类供应商都会有一个默认端点。你也可以新增多个命名端点，例如：

- OpenAI 官方。
- 公司中转。
- 自己的兼容网关。
- 不同账号或不同额度池。

每个端点可以单独保存：

- 名称。
- API Key。
- API Base URL。
- 通过 **获取模型列表** 弹窗纳管的可用模型列表。
- 自定义模型 ID。
- 是否默认。

在高级选项里选择供应商时，会显示这些命名端点，而不是只显示抽象供应商名称。当前图片生成/编辑仍保留一个 **图片端点兼容入口**；它保存后会同步到 **供应商与模型** 中的端点和模型目录，避免旧配置和新目录割裂。

### 模型能力检视

**模型能力检视** 已下沉到 **供应商与模型** 内，不再是系统设置主菜单里的并列入口，也不再承担视频端点新增和凭证管理。它用于跨端点检查模型目录：

- 按供应商、端点、任务能力、来源和状态筛选模型。
- 查看模型来自预置、远端读取还是自定义添加。
- 标记模型是否启用。
- 覆盖模型能力，例如图像生成、图像编辑、图生文、提示词润色、批量规划、文生视频和图生视频。

## 模型列表管理

图片端点、图生文兼容端点和视频供应商端点都使用同一套 **获取模型列表** 弹窗管理模型列表，不再在设置页里平铺模型勾选或使用单选下拉添加。

点击 **获取模型列表** 后会打开弹窗并尝试读取远端模型。弹窗内支持：

- 搜索模型 ID、显示名、厂商和能力。
- 多选模型。
- 全选当前筛选结果。
- 清空当前筛选结果。
- 已经纳管的模型自动选中。
- 手动添加隐藏模型 ID、显示名和厂商。

端点支持读取模型列表时，远端发现结果会先进入模型目录；只有用户在弹窗里应用选择后，模型才写入该端点的纳管白名单。读取模型支持自建的内网、中转和局域网 Base URL，不要求目标地址是公网可达服务。

添加后，它会出现在高级选项的模型列表中。你还可以在 **模型能力检视** 或 **自定义模型能力覆盖** 中调整它的能力：

- 是否支持编辑。
- 是否支持蒙版。
- 是否支持自定义尺寸。
- 是否支持质量、格式、背景、审核、压缩。
- 是否支持流式预览。
- 默认尺寸和尺寸预设。

## Seedream / 火山方舟

Seedream 走火山方舟的 OpenAI Images 兼容协议。Base URL 应填写到兼容接口根路径，例如：

- `https://ark.cn-beijing.volces.com/api/v3`

应用会在 Web 服务器中转、桌面端 Tauri Rust 中转和支持 CORS 的客户端直连模式下请求：

- `POST {Base URL}/images/generations`

注意不要把 Coding Plan / Agent Plan 工具接口当成图片生成接口填写。`https://ark.cn-beijing.volces.com/api/plan` 和 `https://ark.cn-beijing.volces.com/api/plan/v3` 属于 Agent Plan 工具协议，模型可能在该套餐里可见，但并不代表它支持 OpenAI Images 生图请求。若要使用 `doubao-seedream-5-0-260128`，需要使用可访问方舟图片 OpenAI 兼容接口的 API Key，并选择 Seedream 供应商或在 Seedream 端点的可用模型中添加该模型 ID。

## 图生文与多模态模型

**图生文与多模态** 现在是任务设置页。它负责选择图生文端点与模型、任务类型、视觉 detail、输出格式、流式输出、结构化输出、最大输出 Token、系统提示词和历史开关。

图生文模型只来自 **供应商与模型 -> 供应商端点管理** 中已经存在的 OpenAI/OpenAI 兼容或 Anthropic/Anthropic 兼容端点。先选择端点，再点击“尚未选择模型/已选模型”状态框打开模型列表弹窗，弹窗会读取该端点的最新模型列表并以单选方式绑定 `vision.text` 默认模型。未显式选择端点和模型时，图生文提交会提示先完成选择，不会自动使用内置默认模型。

端点新增、凭证编辑、Base URL 和多模型白名单都统一回到 **供应商端点管理**。如果远端模型没有被自动识别为图生文能力，选择绑定时会补充 `vision.text` 能力；也可以在 **供应商与模型 -> 模型能力检视** 中继续校正能力标注。

## 视频供应商与模型

新增视频端点应从 **供应商与模型 -> 供应商端点管理** 进入，并选择视频供应商模板，而不是只使用 OpenAI Compatible。当前模板包括 OpenAI Sora、Google Veo、Runway、Luma、MiniMax、Kling、BytePlus ModelArk、DashScope / Wan、Tencent Hunyuan、Tencent TokenHub、fal.ai 和 xAI。

**视频端点管理** 是分类管理视图。视频供应商端点按模板拆成独立可折叠区块，每个供应商端点都复用统一端点卡片来维护凭证、Base URL 和模型列表。新增视频端点后需要继续点击 **获取模型列表**，在弹窗里多选模型、全选当前结果或手动添加隐藏模型；不支持自动读取的协议也会打开同一个弹窗用于手动添加。空视频端点会显示为待补模型，不会进入视频生成模型下拉。

只有真实已实现适配器的模型会自动成为文生视频 / 图生视频默认候选。协议预留或适配器待实现的供应商可以保存端点和模型，方便后续接入，但提交视频任务前会被阻止。

## 运行与存储

这里控制工作台运行方式：

- **API 连接模式**：服务器中转或客户端直连。
- **最大并发任务数**：控制同时执行多少生成/编辑任务。
- **提示词历史数量**：限制最近提示词保存数量。
- **图片存储模式**：自动、文件系统或 IndexedDB。
- **图片存储路径**：桌面端可填写本机目录。

## 服务器中转与客户端直连

| 模式       | 数据流                             | 适合场景                                 |
| ---------- | ---------------------------------- | ---------------------------------------- |
| 服务器中转 | 浏览器 -> 应用服务器 -> 模型供应商 | 默认推荐，API Key 不直接暴露给浏览器请求 |
| 客户端直连 | 浏览器 -> 模型供应商               | 第三方中转或自托管端点支持 CORS 时使用   |

直连模式要求目标端点支持 CORS。直连时 API Key 会出现在浏览器网络请求中，不适合公共共享机器。

## 提示词润色配置

**提示词润色配置** 只管理单条 prompt 润色。润色模型需要先从 **供应商与模型 -> 供应商端点管理** 已存在的 OpenAI/OpenAI 兼容或 Anthropic/Anthropic 兼容端点中选择供应商端点，再点击“尚未选择模型/已选模型”状态框打开模型列表弹窗，读取最新模型并单选绑定；未选择时不会自动使用任何模型。

润色只支持 OpenAI/OpenAI 兼容和 Anthropic/Anthropic 兼容端点，不要求端点来自指定的文本端点模板。旧的独立润色 API Key、Base URL 和模型 ID 配置已移除；如果还没有可用端点，润色设置会提示先跳转到供应商端点管理添加端点。

可以配置：

- 润色模型选择。
- 是否启用思考模式。
- 思考强度与参数格式。
- 默认内置预设。
- 自定义润色提示词。
- 润色下拉顺序。

## 批量配置

**批量配置** 独立管理批量规划工作流，避免把批量模型和规划提示词混在提示词润色里。批量规划模型同样复用 **供应商与模型 -> 供应商端点管理** 中的 OpenAI/OpenAI 兼容或 Anthropic/Anthropic 兼容端点，但保存为独立的 `prompt.batchPlan` 任务默认模型，不会自动继承润色模型。

可以配置：

- 批量规划模型选择。
- 默认规划方式，例如 AI 自动、内容拆分、多版本探索、参考图变体、文本切分或 JSON 导入。
- AI 自动规划的默认系统提示词，可查看、复制、编辑和恢复默认。
- 批量参数润色开关与范围。
- AI 自动最大任务数、固定数量默认值、单次预览最大条目数和确认前提醒阈值。

文本切分和 JSON 导入不调用 AI，因此即使尚未配置批量规划模型也能生成批量预览；AI 类规划方式会提示先到批量配置中选择模型。

## 密码验证

部署给多人访问时，可以设置 `APP_PASSWORD`。设置后，提交请求前需要输入密码。

注意：客户端直连不经过服务器，因此不会触发 `APP_PASSWORD` 验证。

## 桌面端设置

桌面端基于 Tauri，可以使用 Rust 中转代理处理部分 Web 端容易遇到的限制，例如对象存储 CORS。

桌面设置包括：

- 桌面代理模式。
- 代理地址。
- 调试模式。
- 图片存储路径。

更多安装和打包说明见 [安装、部署与桌面端](./desktop-and-deployment.md)。
