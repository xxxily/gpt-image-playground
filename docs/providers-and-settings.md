# 供应商与系统设置

系统设置是把 GPT Image Playground 变成长期工作台的关键。这里可以配置供应商端点、自定义模型、提示词润色、运行参数、存储、云同步和桌面端选项。

![供应商设置](./images/settings-providers.png)

## 供应商与模型

应用内置四类供应商：

- OpenAI Compatible。
- Google Gemini。
- Seedream。
- SenseNova。

每类供应商都会有一个默认端点。你也可以新增多个命名端点，例如：

- OpenAI 官方。
- 公司中转。
- 自己的兼容网关。
- 不同账号或不同额度池。

每个端点可以单独保存：

- 名称。
- API Key。
- API Base URL。
- 可用模型列表。
- 自定义模型 ID。
- 是否默认。

在高级选项里选择供应商时，会显示这些命名端点，而不是只显示抽象供应商名称。

## 自定义模型

如果某个兼容端点提供了项目未预置的模型，可以在该供应商端点的 **可用模型** 区域添加模型 ID。

端点支持读取模型列表时，也可以先点击 **刷新模型**。刷新成功后，使用 **从已读取模型列表选择** 下拉框选择远端返回的模型，再点击 **添加所选**，不用手动复制模型 ID。
刷新模型支持自建的内网、中转和局域网 Base URL，不要求目标地址是公网可达服务。

添加后，它会出现在高级选项的模型列表中。你还可以在 **自定义模型能力覆盖** 中调整它的能力：

- 是否支持编辑。
- 是否支持蒙版。
- 是否支持自定义尺寸。
- 是否支持质量、格式、背景、审核、压缩。
- 是否支持流式预览。
- 默认尺寸和尺寸预设。

## Seedream / 火山方舟

Seedream 走火山方舟的 OpenAI Images 兼容协议。Base URL 应填写到兼容接口根路径，例如：

- `https://ark.cn-beijing.volces.com/api/v3`

应用会在 Web 服务器中转、桌面端 Tauri Rust 中转和支持 CORS 的客户端直连模式下请求：

- `POST {Base URL}/images/generations`

注意不要把 Coding Plan / Agent Plan 工具接口当成图片生成接口填写。`https://ark.cn-beijing.volces.com/api/plan` 和 `https://ark.cn-beijing.volces.com/api/plan/v3` 属于 Agent Plan 工具协议，模型可能在该套餐里可见，但并不代表它支持 OpenAI Images 生图请求。若要使用 `doubao-seedream-5-0-260128`，需要使用可访问方舟图片 OpenAI 兼容接口的 API Key，并选择 Seedream 供应商或在 Seedream 端点的可用模型中添加该模型 ID。

## 图生文与多模态模型

**图生文与多模态** 现在优先复用统一模型目录中的默认端点和模型，旧的专用端点只作为兼容输入保留。每个图生文端点仍然可以配置 API Key、Base URL、兼容模式和模型 ID。

图生文端点同样支持 **刷新模型** 和 **从已读取模型列表选择**。如果远端模型列表里有可用的多模态模型，选择后点击 **添加所选** 会把模型 ID 写入该端点的模型列表。

设置页里还有一个统一的 **任务默认** 矩阵，可以分别为图生文、提示词润色、文生视频和图生视频挑选默认端点和模型。

## 视频供应商与模型

**统一模型目录** 里的 **新增供应商端点** 可以直接选择视频供应商模板，而不是只使用 OpenAI Compatible。当前模板包括 OpenAI Sora、Google Veo、Runway、Luma、MiniMax、Kling、BytePlus ModelArk、DashScope / Wan、Tencent Hunyuan、Tencent TokenHub、fal.ai 和 xAI。

新增视频端点后需要继续在端点卡片里 **选择模型**。支持远端模型列表的端点会读取模型并显示可多选列表，支持搜索、全选当前结果、清空选择和批量添加；不支持自动读取的端点仍然保留手动模型 ID 添加。空视频端点会显示为待补模型，不会进入视频生成模型下拉。

只有真实已实现适配器的模型会自动成为文生视频 / 图生视频默认候选。协议预留或适配器待实现的供应商可以保存端点和模型，方便后续接入，但提交视频任务前会被阻止。

## 运行与存储

这里控制工作台运行方式：

- **API 连接模式**：服务器中转或客户端直连。
- **最大并发任务数**：控制同时执行多少生成/编辑任务。
- **提示词历史数量**：限制最近提示词保存数量。
- **图片存储模式**：自动、文件系统或 IndexedDB。
- **图片存储路径**：桌面端可填写本机目录。

## 服务器中转与客户端直连

| 模式       | 数据流                             | 适合场景                                 |
| ---------- | ---------------------------------- | ---------------------------------------- |
| 服务器中转 | 浏览器 -> 应用服务器 -> 模型供应商 | 默认推荐，API Key 不直接暴露给浏览器请求 |
| 客户端直连 | 浏览器 -> 模型供应商               | 第三方中转或自托管端点支持 CORS 时使用   |

直连模式要求目标端点支持 CORS。直连时 API Key 会出现在浏览器网络请求中，不适合公共共享机器。

## 提示词润色配置

润色默认从统一模型目录里选择文本模型；如果需要临时连接，仍可以在高级区域单独填写 API Key 和 Base URL。

可以配置：

- 润色默认模型。
- 是否启用思考模式。
- 默认内置预设。
- 自定义润色提示词。
- 润色下拉顺序。

## 密码验证

部署给多人访问时，可以设置 `APP_PASSWORD`。设置后，提交请求前需要输入密码。

注意：客户端直连不经过服务器，因此不会触发 `APP_PASSWORD` 验证。

## 桌面端设置

桌面端基于 Tauri，可以使用 Rust 中转代理处理部分 Web 端容易遇到的限制，例如对象存储 CORS。

桌面设置包括：

- 桌面代理模式。
- 代理地址。
- 调试模式。
- 图片存储路径。

更多安装和打包说明见 [安装、部署与桌面端](./desktop-and-deployment.md)。
