## 问题与解决

| 问题                                                                                            | 解决办法                                                                                                                                       | 剩余风险                                                                                       |
| ----------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| 浏览器出于 CORS 会把响应细节隐藏成 `TypeError: Failed to fetch`，原 UI 把它当普通网络错误展示。 | 复用 `isLikelyWebDirectAccessError`，将这类错误映射为 `cors`，并展示跨域受限、桌面端或放行当前站点的建议。                                     | 浏览器不会暴露所有跨域细节，只能基于典型直连失败特征判断。                                     |
| React state 异步更新会让“添加并保存”保存旧的端点/模型目录。                                     | `createUnifiedProviderEndpointAddition` 先构造完整快照，`saveSettings({ endpointAddition })` 直接使用快照保存。                                | 其它旧版 provider instance 编辑路径仍依赖现有 `rebuildProviderEndpoints`，本次未做大范围重构。 |
| 新增端点时模型目录 normalize 可能清理旧任务默认模型。                                           | 保存和新增端点时把当前 `modelTaskDefaultCatalogEntryIds` 一起传入 normalize，并用测试锁定 `prompt.polish`、`prompt.batchPlan`、`vision.text`。 | 端点被删除、模型被禁用或能力不匹配时，默认模型仍会按既有规则失效。                             |
| UI 自动化环境没有项目内 Playwright 依赖，Browser 插件的 Node 执行入口也未暴露。                 | 使用系统 Chrome headless + CDP 执行本地交互验证，并在结束后关闭端口、删除临时 Chrome profile。                                                 | 该验证是手写最小 CDP 脚本，不替代完整端到端测试套件。                                          |
