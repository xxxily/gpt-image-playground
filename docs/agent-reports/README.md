# Agent 任务报告

这里保存每次由 agent 执行并改变仓库状态的任务报告，用于复盘、接力、验收和后续评分。报告不是替代需求文档，而是记录“本次实际做到了什么、怎么验证、还剩什么风险”。

## 适用范围

- 修改代码、文档、配置、构建脚本、发布流程或运行环境的任务，必须写入报告。
- 继续处理已有需求或阶段任务时，优先更新同一份报告；只有任务边界明显不同才新建报告。
- 纯问答、只读排查且没有改变仓库状态的任务，可以不建报告，但最终回复需要说明未建报告的原因。

## 文件命名

使用当前日期和简短英文 slug：

```text
docs/agent-reports/YYYY-MM-DD-short-task-slug.md
```

如果任务直接对应 `docs/requirements/` 中的某份需求文档，slug 应尽量复用需求主题，方便检索。

## 语言规则

- 报告正文、标题、章节名、表格字段和最终回复，应跟随用户本轮请求的主要语言，除非用户明确指定其他语言。
- 不要固定使用英文或中文模板。英文用户写英文报告，中文用户写中文报告；用户混合表达时，以主要意图语言为准，必要时保留原文术语。
- 文件路径、命令、代码标识、API 名称、模型名、commit hash、精确需求文档标题等技术标识可以保留原文，避免翻译后影响检索或准确性。
- 状态可以使用本地化表达，但建议同时保留规范枚举，便于后续检索和评分，例如 `已完成 (Completed)`、`部分完成 (Partial)`、`受阻 (Blocked)`。

## 必填结构

每份报告至少包含以下语义内容。下面是中文请求的示例；如果用户主要使用其他语言，应把标题和表格字段翻译成对应语言。

```markdown
# <任务标题>

| 字段     | 内容                                                     |
| -------- | -------------------------------------------------------- |
| 日期     | YYYY-MM-DD                                               |
| 状态     | 已完成 (Completed) / 部分完成 (Partial) / 受阻 (Blocked) |
| 相关请求 | <用户请求或需求文档>                                     |
| 相关文档 | <相关需求、计划、说明文档链接>                           |
| 改动范围 | <代码/文档/配置/运行时范围>                              |
| 提交状态 | <未提交 / 已提交 commit hash / 不适用>                   |

## 范围核对

| 请求目标 | 实际结果 | 证据 | 状态                                                                    |
| -------- | -------- | ---- | ----------------------------------------------------------------------- |
| ...      | ...      | ...  | 已完成 (Completed) / 部分完成 (Partial) / 受阻 (Blocked) / 不适用 (N/A) |

## 问题与解决

| 问题 | 解决办法 | 剩余风险 |
| ---- | -------- | -------- |
| ...  | ...      | ...      |

## 验证

| 检查项 | 命令或场景 | 结果 |
| ------ | ---------- | ---- |
| ...    | ...        | ...  |

## 后续建议

- ...
```

## 编写要求

- 不允许只写“已完成”。必须能从报告中看出需求覆盖率、实际改动、验证证据和剩余风险。
- 不允许因为模板方便而固定使用英文或中文；报告语言必须按上面的语言规则选择。
- 对未完成、跳过或无法验证的项目要显式标记，并说明原因。
- 涉及 UI 时记录桌面/移动、浅色/深色、浏览器场景；没有执行时说明原因。
- 涉及 Web 与 Tauri 分支时分别记录验证或不适用原因。
- 不写入 API Key、Token、密码、带凭证 URL、用户隐私数据或无关的本机敏感信息。
- 最终回复中必须给出本次报告路径，方便后续 agent 接力。
