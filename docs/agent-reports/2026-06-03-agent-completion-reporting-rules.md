## 问题与解决

| 问题                                                         | 解决办法                                                                             | 剩余风险                                                            |
| ------------------------------------------------------------ | ------------------------------------------------------------------------------------ | ------------------------------------------------------------------- |
| 原规则只要求简要 handoff，仍可能允许 agent 泛泛宣称完成。    | 增加显式完成汇报规则，并要求在宣称完成前按原始请求和需求文档逐项核对。               | 规则本身不能机械保证执行质量，但报告会让遗漏更容易被审计。          |
| 任务报告存储位置不存在。                                     | 新建 `docs/agent-reports/` 并记录报告结构。                                          | 后续接力任务应更新既有报告，避免重复建档。                          |
| 初版报告模板和本次报告正文偏英文，不符合用户补充的语言要求。 | 增加语言规则：报告正文、章节、表格字段和最终回复跟随用户主要语言；技术标识保留原文。 | 混合语言请求仍需要 agent 判断主要语言；必要时应按用户显式要求处理。 |
| 工作树存在无关源码修改和未跟踪文件。                         | 未检查、未修改、未回退这些无关源码文件。                                             | 后续提交前需要把本次文档改动和无关源码改动分开处理。                |
