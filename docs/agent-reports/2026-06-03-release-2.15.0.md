## 问题与解决

| 问题                                           | 解决办法                                                                                      | 剩余风险                                                                         |
| ---------------------------------------------- | --------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| 标准流程默认包含 `142` 与 `129` 两台服务器部署 | 本次遵循用户最新约束，跳过 `142`，仅执行 `129` 部署和 `img-playground.anzz.site` 访问验证      | `142` 对应域名不会在本次发版得到更新；服务器恢复后需要按 `v2.15.0` tag 单独补发 |
| 版本发布包含多运行时构建                       | 按发布规范执行 npm、Next、桌面静态 build 与 Rust 侧校验，失败项会在本报告记录                  | Rust audit 通过但报告 19 条允许的上游依赖警告，主要来自 Tauri/wry/GTK/urlpattern/rand 依赖链 |
| 发布流程会读取环境文件状态                     | 只运行 `release:env-check` 报告 `.env*` 文件存在与 Git 跟踪状态，不写入或展示任何 secret 内容 | 无                                                                               |
| 本地 `.env.local` 缺少 `BETTER_AUTH_SECRET`    | 部署前检查 129 持久化 `.env` 中该 key 存在，并把远端现有值作为本次部署进程环境变量传入；部署后再次确认 key 仍存在 | 远端 `.env` 中有一行不适合直接 shell source 的 URL 值，读取 secret 时出现过非阻塞报错；后续可改用 dotenv parser 提取 |
