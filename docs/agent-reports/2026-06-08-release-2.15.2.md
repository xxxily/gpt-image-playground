## 问题与解决

| 问题 | 解决办法 | 剩余风险 |
| --- | --- | --- |
| 标准流程默认要求工作区干净 | 当前存在发布前已有的无关社区推广文档改动；本次仅暂存和提交发布相关文件，避免混入无关变更。 | `git status` 会继续显示这些无关未提交文件，不代表发布提交包含它们 |
| 标准流程默认包含 `142` 与 `129` 两台服务器部署 | 本次遵循用户最新约束，跳过 `142`，仅执行 `129` 部署和 `img-playground.anzz.site` 访问验证。 | `142` 对应域名不会在本次发版得到更新；服务器恢复后可按 `v2.15.2` tag 单独补发 |
| 本地 `.env.production` 缺少生产 auth secret | 部署 `129` 时从远程共享环境读取已有生产密钥并通过环境变量传入 `scripts/deploy-129.sh`；日志和报告只记录脱敏占位，不记录密钥值。 | 后续发版仍需要确保 `BETTER_AUTH_SECRET` 和 `ADMIN_BOOTSTRAP_SECRET` 不被空值覆盖 |
| `src-tauri/Cargo.lock` 被 `.gitignore` 忽略 | 本地同步到 `2.15.2` 并纳入版本一致性检查；发布提交不包含该忽略文件。 | 如果 CI 未来改为强制跟踪 Cargo lock，需要先调整 `.gitignore` 和提交策略 |
| GitHub Actions annotations 包含 Node 20 deprecation 与 checkout git 128 warning | 逐 job 核对 conclusion，所有 job 均为 `success`；这些 annotations 未导致构建失败。 | GitHub 将在 2026-06-16 起默认切到 Node 24，后续可评估 Actions 版本和 runner 环境 |
| macOS 未配置 Apple 签名/公证 secrets | workflow 按发布规范上传 unsigned/unnotarized DMG，并在 Release notes 中包含 Gatekeeper 用户侧说明。 | macOS 用户首次打开可能需要按说明解除 quarantine；配置 Apple secrets 后可消除该风险 |
