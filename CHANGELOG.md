# 更新日志

## v2.6.1 - 2026-05-07

### 重点更新

- 系统设置中“供应商 API 配置”改为独立内页入口，点击后进入专属配置页面，各供应商 API Key 和 Base URL 分别展开配置，不再挤在一个大折叠块里。
- 关于弹窗精简，移除面向开发者的口语化描述，保留应用信息与更新检查。
- 桌面端专属引导：Web 运行时自动隐藏桌面代理/调试等配置项，改用统一提示引导用户下载桌面端；配置跨域或直连请求失败时同样给出明确跳转引导。

### 交互体验

- 对话框移动端自适应高宽：小屏幕下弹窗自动撑满视窗并保留安全区域底部间距，关闭按钮扩大至 44×44 触摸热区，彻底解决移动端上下内容被遮挡无法操作的问题。
- 系统设置取消操作增加未保存确认，避免误点直接流失已填写的 API 配置。

### 稳定性与错误处理

- 修复提示词润色旧配置隐式覆盖内置预设的问题，现在只有用户手动选中“已保存自定义”或“临时自定义”才会覆盖。
- 共享桌面端引导文案模块，统一连接策略、任务执行器、提示词润色等模块中的错误提示语，消除分散维护导致的文案不一致。
- 增加桌面引导、连接策略和润色配置相关测试覆盖。

### 文档与发布

- 同步更新 Web、package-lock、Tauri 和 Cargo 版本号到 `2.6.1`，用于触发 `v2.6.1` GitHub Release 与桌面端构建。

### 升级注意事项

- 本次为功能改进版本，无需额外迁移操作。系统设置中的历史配置会自动保留，供应商 API 配置页面进入路径略有变化但不影响功能使用。

## v2.6.0 - 2026-05-06

### 重点更新

- 提示词润色升级为多预设选择体验，内置均衡、简洁、编辑优化、电影感、写实摄影、插画、商业产品和极简构图等多种场景化润色规则。
- 润色按钮现在会像历史和模板一样弹出选择框，支持搜索内置预设、按源图自动推荐编辑优化预设，并可输入仅本次生效的临时自定义润色提示词。
- 系统设置中的润色配置重新梳理为“内置预设”“已保存自定义”“临时自定义”三种独立来源：内置预设只读可查看，已保存自定义可编辑并在下拉中手动选择，临时自定义不会覆盖系统设置。
- 桌面端新增更新检查、应用内通知和更多代理配置控制，Rust/Tauri 代理能力继续扩展到提示词润色、Gemini/OpenAI 配置和桌面直连辅助场景。

### 交互体验

- 修复亮色主题下润色选择框、历史下拉、模板快捷命令和分享按钮 hover 状态的文字/图标对比度问题，浅色界面下更容易阅读。
- 统一提示词区域下拉层关闭行为：点击输入框、空白区域或切换选择后，润色、历史和模板下拉都会按预期关闭，减少残留浮层干扰。
- 设置面板会展示当前内置润色预设的完整系统提示词、分类和只读状态，用户能清楚知道每个预设如何工作。
- 桌面端全屏预览补充缩放手势修复，历史记录相关操作在桌面环境中更稳定。

### 稳定性与错误处理

- 提示词润色请求不再让已保存自定义提示词隐式覆盖选中的内置预设，只有用户在下拉中明确选择“已保存自定义”或“临时自定义”时才会作为本次请求覆盖。
- 增加提示词润色预设、系统提示词解析、桌面配置和代理命令相关测试，覆盖旧配置默认值、大小写空格兼容和桌面代理边界。
- 桌面导出构建保留必要 API Routes，避免桌面构建过程误删服务端接口导致运行时异常。

### 文档与发布

- 同步更新 Web、package-lock、Tauri 和 Cargo 版本号到 `2.6.0`，用于触发 `v2.6.0` GitHub Release 与桌面端构建。

### 升级注意事项

- 如此前在系统设置里填写过自定义润色提示词，升级后它会作为“已保存自定义”选项出现在润色下拉中，但不会再自动覆盖内置预设；需要使用时请在下拉中手动选择。

## v2.5.0 - 2026-05-06

### 重点更新

- 桌面端服务器中转模式接入 Rust 请求代理，图片生成、编辑和远程图片代理不再依赖 Node 服务端，降低 Tauri 安装包体积压力。
- 桌面端历史图片、远程图片发送到编辑和本地图片下载改由 Tauri IPC 调用 Rust 处理，保持 Web 端 Next API Routes 行为不变。
- OpenAI 桌面端流式预览改为通过 Tauri Channel 转发图片生成事件，支持 partial image progress 与完成事件回传。

### 稳定性与错误处理

- 增加远程图片代理的 URL、重定向、DNS 私网地址和响应体大小校验，降低 SSRF 与异常大文件风险。
- 增加桌面端上传文件大小限制、本地图片文件名校验和删除路径校验，避免路径穿越和误删非生成图片。
- 补充 Rust SSE 解析、远程/本地图片处理、OpenAI 流式代理和桌面 IPC 包装测试，覆盖畸形事件、`[DONE]` sentinel、上传大小限制与 Channel 消息转发。

### 文档与发布

- 更新 Tauri Rust 请求代理阶段规划文档，记录 Phase 3/4 实现范围、验证结果、5 路审查结论和后续非阻塞跟进项。
- 同步 Web、package-lock、Tauri 和 Cargo 版本号到 `2.5.0`，用于触发 `v2.5.0` 桌面端 GitHub Release 构建。

### 升级注意事项

- 本次先部署到 `142` 测试环境验证 Docker 构建、GitHub Actions 桌面构建和人工使用体验；确认正常后再按需部署 `129` 生产环境。

## v2.4.0 - 2026-05-05

### 重点更新

- 新增 SenseNova U1 Fast Provider 支持，可接入商汤 SenseNova 图像生成服务，并支持 2K 分辨率、常用比例预设和专属响应参数。
- 新增 Seedream Provider 支持，可接入火山方舟 Doubao Seedream 模型，并提供 1K/2K 分辨率、PNG/JPEG 输出、水印开关、组图生成、Seed、Guidance Scale、提示词优化和联网搜索等专属参数。
- 完善 Provider-aware 参数控制架构：编辑表单会根据当前供应商自动展示对应参数，OpenAI 保留质量、输出格式、背景、审核等通用控制，SenseNova 与 Seedream 展示各自文档化的高级选项。
- 系统设置面板新增 SenseNova/Seedream 配置区块，API Key 与 Base URL 支持显示/隐藏切换，配置优先级保持为 UI 设置 > `.env` > 系统默认。

### Provider 生态

- 新增 OpenAI-compatible Provider Adapter，统一多供应商图像请求适配层，为后续扩展更多兼容 OpenAI API 的图像服务打基础。
- 新增 Provider Credential Config 与 Provider Options Helpers，集中处理各供应商默认端点、运行时凭证、模型能力标记、请求参数合并和参数校验。
- 模型注册表接入多供应商模型，模型选择流程先区分 Provider，再展示对应模型，降低模型列表随供应商增加后的筛选成本。
- 分享链接支持携带 Provider 配置，接收者可选择仅本次使用、保存到本地设置或忽略分享中的 API 配置。

### 交互体验

- 优化移动端提示词模板弹窗排版，让搜索、模板列表和底部操作区在小屏幕上更紧凑。
- 历史记录移动端操作按钮改为更省空间的布局，常用动作在窄屏下更容易触达。
- 精简图片表单辅助说明和关于弹窗文案，减少非关键文本对主流程的干扰。

### 稳定性与错误处理

- 强化客户端直连模式错误提示，API 请求失败时返回更清晰的错误类型和说明，便于定位 CORS、认证或端点配置问题。
- 使用本地系统字体令牌，减少外部字体依赖和首屏网络请求，提升加载稳定性。

### 文档与发布

- README 补充 SenseNova/Seedream 配置说明，新增相关环境变量示例和默认 Base URL 说明。
- `.env.example` 新增 SenseNova 与 Seedream 配置模板，便于部署前快速填写供应商凭证。
- 同步更新 Web 与桌面端版本号到 `2.4.0`，确保 `v2.4.0` 标签触发的 GitHub Release 和桌面端构建流程通过版本一致性校验。
- 新增根目录发布规范文档，沉淀版本更新、tag 推送、GitHub Actions 构建和双服务器部署的标准流程。

### 升级注意事项

- 如需使用 SenseNova 或 Seedream，请在系统设置或 `.env.local` 中配置对应 API Key；Base URL 留空时会使用系统默认端点。
- Provider-aware 参数面板会随供应商切换动态变化，升级后建议验证 OpenAI、SenseNova、Seedream 的模型选择、参数展示、生成和分享链路。
- 如已启用 `CLIENT_DIRECT_LINK_PRIORITY=true`，非官方 Provider Base URL 会继续触发客户端直连优先策略，请确认目标服务支持浏览器 CORS。

- 新增提示词模板与斜杠命令系统，可通过 `/` 快速插入常用图片生成/编辑模板，并支持键盘导航。
- 新增并发任务系统，支持任务持久化、任务卡片、任务追踪器和毫秒级耗时展示。
- 新增 UI 系统设置面板，支持 API Key、Base URL、图片存储模式、连接模式和自定义模型配置热更新。

### 交互体验

- 历史图片支持双击全屏预览，并可从历史记录或预览中快速“发送到编辑”。
- 图片查看器支持点击缩放、滚轮缩放、双指缩放、拖拽移动、点击外部关闭和 ESC 退出。
- 编辑模式支持全局剪贴板粘贴和全局拖拽上传，图片输入路径更顺手。
- 设置页、主工作区、历史面板和辅助对话框完成移动端与亮色/暗色主题适配。

### 稳定性与错误处理

- 强化 API 错误处理与连接策略，新增环境变量示例，第三方中转/直连场景提示更清晰。
- 修复取消任务时缺失 AbortController、设置保存/重置后的状态同步、页面缩放误触和右键拖拽误触等问题。
- 修复缩放查看器拉伸变形、点击外部关闭、垂直居中和布局标签换行等 UI 问题。

### 国际化与文档

- 完成整体 UI 简体中文翻译，并修复中文字符串引号问题。
- README 补充连接模式、系统设置和构建说明。

### 构建与发布

- 新增跨平台 Tauri 构建工作流，并在 v2.0.0 中升级为 tag 驱动发布：推送 `v*` 标签后会校验版本、生成 GitHub Release 描述并上传桌面安装包。
- 构建流程新增版本一致性检查、Lint、TypeScript 类型检查、Web 构建预检和工作流产物上传，减少发布失败和描述过简的问题。

### 升级注意事项

- 生成/编辑表单已统一为新的工作流，历史使用习惯可能略有变化。
- 图像任务统一经过 Provider-aware 执行层，建议同时验证 OpenAI 与 Gemini 配置。
- 新增任务持久化与主题系统后，如遇到旧配置显示异常，可在系统设置中重置相关配置。
