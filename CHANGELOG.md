## v2.15.3 - 2026-06-16

### 重点更新

- 服务端生成任务接管完成 P0 Phase 0-6：新增独立 `gpt-image-task-service` 骨架、Hatchet adapter entrypoint、任务服务 HTTP contract、当前 App 管理端接入、Web 用户侧提交/恢复/结果导入、任务服务不可用 fallback、安全审计、自动重试和故障演练能力。
- 供应商端点管理新增真实可用的“测试连接”入口；浏览器直连遇到目标端点 CORS 限制时，会明确提示“跨域策略拦截”，并建议改用桌面端或让目标网站/中转服务允许当前站点跨域访问，不再只暴露 `Failed to fetch`。
- 新增供应商端点草稿保存确认：用户填写名称、API Key 或 Base URL 后直接保存配置时，可选择继续编辑、不添加并保存，或添加当前端点并保存。

- 图生文直连失败也会识别典型浏览器直连/CORS 错误，并追加桌面端或 CORS 放行建议。
- 将 `@tailwindcss/vite` 归入构建期依赖，并用 npm overrides 锁定安全版 `vite` / `esbuild`，让生产依赖审计重新回到 0 vulnerabilities。
- 当前 App 的 TypeScript/Vitest 校验明确排除独立 `gpt-image-task-service` 子项目，避免根依赖环境误扫子项目 Hatchet SDK；独立服务继续使用自己的 Node test runner 验证。
- 补充供应商连接测试、模型绑定、图生文错误翻译、任务服务运行时、后台安全、故障演练和 Web 恢复等回归测试。

### 文档与发布

- 更新供应商设置文档和任务报告，记录端点草稿保存确认、CORS 直连限制提示和默认模型保留验证。

## v2.14.2 - 2026-05-31

### 重点更新

- 展示内容后台新增展示组约束体系：全局展示组和分享展示组现在可以配置“所有域名均显示”或“仅指定域名显示”，公共展示接口会按实际请求域名筛选展示组，并保留后续扩展每日时间段、语言、渠道等约束的底层模型。
- 展示组列表新增约束摘要搜索，域名约束编辑支持添加当前域名、查看精确/通配类型并删除单条规则；域名专属展示组没有有效素材时会继续使用匹配的全域兜底组。

### 稳定性与错误处理

- 公共展示接口现在只使用服务端解析到的请求 Host 进行域名匹配，不接受公开 query 参数伪造域名；未知或损坏的约束配置会按不匹配处理，避免受限展示组被错误放大到所有域名。
- 展示组保存前会归一化并校验已知约束 payload，历史无约束数据继续按全域展示处理，分享展示组域名不匹配时会安全降级到当前域名匹配的全局展示组。
